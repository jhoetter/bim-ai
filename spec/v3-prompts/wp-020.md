## Verbatim CMD-V3-02 tracker entry (§6 T9, line 3176)

**CMD-V3-02 — Assumption log primitive**

_Source: §L Locked Decisions ("agent MUST log assumptions JSON before apply"); v2 SKB-08 `AgentAssumptionElem` carried forward as a kernel-level structured field. Reframes from "agent assumption log feature" to "kernel-validated structured field on every bundle."_

**Status.** `now`.
**Scope.** Every CMD-V3-01 bundle MUST carry a non-empty `assumptions: AssumptionEntry[]`. The kernel does **not** interpret semantics — it does not know whether `confidence: 0.7` is calibrated, does not validate that `source: "brief-line-12"` actually points at the brief. The kernel **only** validates _structure_: required fields, correct types, non-empty array, unique keys.

This operationalises §L's "assumptions-first" mandate. The agent fills semantics; the kernel enforces shape. A human script fills the field the same way — the contract is structural, not AI-specific. Every entry's `contestable` flag drives whether the Inspector exposes an "edit + re-run" affordance.

**Data model.**
```ts
type AssumptionEntry = {
  key: string;                                  // 'powder-room-area-m2'
  value: string | number | boolean;             // 3.0
  confidence: number;                           // [0..1]; uncalibrated by the kernel
  source: string;                               // 'brief-line-12' | 'sketch-region-3' | 'agent-default'
  contestable?: boolean;                        // default true
  evidence?: string;                            // optional free-text justification
};
```
**Engine.** Validation in `app/bim_ai/cmd/apply_bundle.py`. Every element the bundle creates gets an `agentTrace` field linking back to the bundle's assumptions array. Activity stream renders assumption entries inline (cross-link to T3). Audit log records every assumption verbatim.
**UI.** Inspector for any element with `agentTrace` shows an "Assumption log" section: list of entries; for `contestable: true` entries, an "Edit + re-run" button that opens a small form (key + value), submits a new bundle with the corrected value via CMD-V3-01.
**Acceptance.** (a) Bundle without `assumptions` → rejected with `assumption_log_required`. (b) Bundle with malformed entry (missing `confidence`) → rejected with `assumption_log_malformed` + which-entry pointer. (c) Bundle with valid 5-entry log → apply lands; Inspector for the new powder room shows 5 entries; editing "area: 3.0" → "1.8" and clicking re-run submits a new bundle. (d) Activity stream filterable by "Show entries with assumption log" (any caller, not just AI).
**Effort.** S — 4 days.
**Cross-theme references.** **API-V3-01.** **CMD-V3-01.** **T3 (activity stream renders assumption entries).** **§L (operational instrument of the locked decision).**

## Implementation tasks

### 1. `AgentTrace` model — `app/bim_ai/cmd/types.py`

Add an `AgentTrace` model that links kernel elements back to the bundle's assumption entries:

```python
class AgentTrace(BaseModel):
    model_config = ConfigDict(populate_by_name=True)
    bundle_id: str = Field(alias="bundleId")
    # Indices into the bundle's assumptions array; empty = all assumptions apply
    assumption_keys: list[str] = Field(default_factory=list, alias="assumptionKeys")
    applied_at: str = Field(alias="appliedAt")  # ISO-8601 UTC timestamp
```

This model is the payload stored in each element's `agent_trace` field (see task 2).

### 2. `agent_trace` field on kernel elements (`elements.py`)

Add an optional `agent_trace` field to the base element class (or a shared mixin) so every element can carry provenance:

```python
agent_trace: "AgentTrace | None" = Field(default=None, alias="agentTrace")
```

If `elements.py` uses a shared `BaseElement` or `ElementBase` class, add the field there. If not, add it to the most common concrete types (at minimum `WallElem`, `FloorElem`, `RoofElem`, `DoorElem`, `WindowElem`, `StairElem`, `RailingElem`). The field defaults to `None` — all existing elements round-trip unchanged.

Import `AgentTrace` at the top of `elements.py` from `bim_ai.cmd.types`; use `TYPE_CHECKING` guard to avoid circular imports if needed.

### 3. Attach `agent_trace` in `apply_bundle` (`app/bim_ai/cmd/apply_bundle.py`)

After `engine.try_commit_bundle(doc, bundle.commands)` succeeds, iterate over the newly created/modified elements and attach `AgentTrace`:

```python
from datetime import datetime, timezone

def _attach_agent_traces(
    new_elements: dict,
    old_elements: dict,
    bundle: CommandBundle,
    bundle_id: str,
) -> None:
    """Stamp every element that is new or modified by this bundle with an AgentTrace."""
    now_iso = datetime.now(timezone.utc).isoformat()
    assumption_keys = [entry.key for entry in bundle.assumptions]
    for eid, elem in new_elements.items():
        if eid not in old_elements or old_elements[eid] != elem:
            # This element was created or modified by the bundle
            new_elements[eid] = elem.model_copy(update={
                "agent_trace": AgentTrace(
                    bundle_id=bundle_id,
                    assumption_keys=assumption_keys,
                    applied_at=now_iso,
                )
            })
```

Call `_attach_agent_traces` only when `mode == "commit"` and `apply_bundle` is returning `applied=True`.

The `bundle_id` can be derived as the SHA-256 of the bundle's JSON (consistent with the `checkpoint_snapshot_id` approach in CMD-V3-01) or as a new UUID generated per apply call. Use UUID4 for simplicity unless the WP-009 code already establishes a bundle id convention.

### 4. Audit log append (`app/bim_ai/cmd/apply_bundle.py`)

When `mode == "commit"` and `applied=True`, write an audit log entry. For v3.0, this is a simple append to a per-model JSONL file:

```python
def _write_assumption_audit(
    model_id: str,
    bundle_id: str,
    assumptions: list[AssumptionEntry],
    applied_at: str,
) -> None:
    """Append an audit record to <data_dir>/models/<model_id>/audit/assumptions.jsonl.

    Creates the directory if it does not exist.
    Each line is a JSON object: { bundleId, appliedAt, assumptions: [...] }.
    """
    import json, os
    audit_dir = os.path.join("data", "models", str(model_id), "audit")
    os.makedirs(audit_dir, exist_ok=True)
    record = {
        "bundleId": bundle_id,
        "appliedAt": applied_at,
        "assumptions": [a.model_dump(by_alias=True) for a in assumptions],
    }
    with open(os.path.join(audit_dir, "assumptions.jsonl"), "a") as f:
        f.write(json.dumps(record) + "\n")
```

Call `_write_assumption_audit` after `_attach_agent_traces`. This write is a best-effort side effect — wrap in a `try/except` so an I/O failure does not roll back the commit.

## Implementation rules

- **No AI in any binary.** `apply_bundle` + `_attach_agent_traces` are fully deterministic. No LLM, no inference.
- **API-V3-01 contract.** The `AssumptionEntry` shape is already registered in `registry.py` by CMD-V3-01. This WP adds `AgentTrace` to the output schema of the `apply-bundle` ToolDescriptor — update the `outputSchema.elements[].agentTrace` field in `registry.py`.
- **Assumptions-first (§L).** The kernel enforces structure; semantics live in the agent. The audit log is append-only — no delete endpoint in v3.
- **Backward compatibility.** `agent_trace=None` on existing elements — zero-migration. The `agentTrace` field is optional everywhere.
- **Pydantic alias discipline.** All new Python fields use `Field(alias="camelCase")` and `populate_by_name=True`.
- **No hex literals in chrome.** `AssumptionLogSection.tsx` must use only CSS token variables.
- **Audit log is best-effort.** I/O failures in `_write_assumption_audit` must not abort the commit — always wrap in `try/except Exception`.
