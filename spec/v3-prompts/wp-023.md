### 2. Status bar layout — 6 slots

The status bar is a 28 px high, full-width bar at the bottom of the canvas. It has 6 named slots:

- Slot 1 (left): model name / save status
- Slot 2: active level / cut-plane indicator
- Slot 3: view type label (Plan | Section | 3D | Sheet)
- Slot 4: active phase
- Slot 5 (keystone): lens dropdown — "Show: All" / "Show: Structure" / etc.
- Slot 6 (right): drift badge (hidden when drift = 0)

If a status bar component already exists in the codebase, extend it to implement this slot layout. If none exists, create `packages/web/src/chrome/StatusBar.tsx`.

The bar must be styled with:
```css
height: 28px;
background: var(--color-surface-strong);
border-radius: var(--radius-canvas);   /* = 0 */
font-size: var(--text-2xs);
```

### 4. Drift badge (slot 6)

The drift badge shows a count of federation drifts from `monitorDriftBadge`:

```tsx
// DriftBadge.tsx — part of StatusBar or a separate file in chrome/
interface DriftBadgeProps {
  driftCount: number;
  onClick: () => void;  // opens the federation panel
}

export function DriftBadge({ driftCount, onClick }: DriftBadgeProps) {
  if (driftCount === 0) return null;
  return (
    <button
      style={{ color: 'var(--color-drift)' }}
      onClick={onClick}
    >
      {driftCount} drift{driftCount !== 1 ? 's' : ''}
    </button>
  );
}
```

The `--color-drift` token must be present in the design system. If it is not yet defined, add it to the CSS token file (locate via `grep -rn "color-drift\|--drift" packages/web/src/`). Use `oklch(65% 0.18 50)` as the initial value (amber warning tone consistent with the bim-ai token palette).

## Implementation rules

- **No hex literals in chrome.** Every colour in `StatusBar.tsx`, `LensDropdown.tsx`, and `DriftBadge.tsx` must use `var(--…)` CSS token references. The ESLint `bim-ai/no-hex-in-chrome` rule is active; CI fails on any literal hex in new `.tsx` files. The single exception is the `--color-drift` token initialisation value in the CSS token file itself — token definitions may use `oklch(…)` or `#` values; component code may not.
- **28 px height is fixed.** Do not use `min-height` or `height: auto`. The bar must be exactly 28 px.
- **`--radius-canvas: 0`.** The bar has no border-radius (sharp edges, flush with the canvas bottom).
- **`--ease-paper` for all transitions.** Lens change, drift badge appear/disappear — all use `transition: opacity 240ms var(--ease-paper)`.
- **Lens dispatch reuses T8 commands.** Do not invent a new store or command. Import and use whatever T8 DSC-V3-02 shipped for lens state.
- **No filter dialog.** The lens dropdown is the complete filter surface. No modal, no sheet overlay. This is the A8 antidote — the whole point is that it replaces the dialog.
- **Drift badge opens federation panel.** The onClick for `DriftBadge` must open the federation/links panel. Locate the existing panel toggle action (search `packages/web/src/` for `federationPanel`, `linksPanel`, or `FederationPanel`) and dispatch it.
- **Pydantic alias discipline** does not apply (this is a pure frontend WP), but follow the same `camelCase` prop naming convention in TypeScript interfaces.
