## Verbatim IMG-V3-01 tracker entry (§6 T9, line 3243)

**IMG-V3-01 — Image-input → structured-layout tool (deterministic CV)**

_Source: R-A §8 (AI tracing image → vector); R-C §2 + §4 (AutoLayout multi-modal input). Reframed entirely from "agent traces sketch via multimodal AI" to "**deterministic CV** tool that produces a structured layout an external agent can read." The wire-format JSON is the contract; the CV implementation is swappable._

**Status.** `now` for the deterministic CV path. `vision` for AI-swappable upgrades.
**Scope.** A deterministic CV tool — line detection, polygon recovery, optional OCR for dimensions — that takes an image and returns a structured layout JSON. **No AI runs inside the binary.** Pipeline:

1. **Preprocessing.** SKB-04 calibrator + SKB-14 edge detection + SKB-07 region colour sampler (v2 primitives).
2. **Polygon recovery.** Hough line detection → contour fitting → room-region segmentation. Pure CV.
3. **Dimension OCR (optional).** Tesseract on labelled strings; rejected gracefully if illegible.
4. **Layout JSON emission.** Returns a `StructuredLayout` round-trippable into TKN-V3-01's `EnvelopeToken[]`.

The output JSON is the wire format an external agent reads. The agent decides how to interpret it — refine with its own vision, call CMD-V3-01 directly, combine with a brief. The CV is the **floor**: a deterministic baseline that always works. AI augmentations are external and optional.

**Data model.** New `StructuredLayout` type designed to be both human-debuggable and TKN-V3-01-compatible:
```ts
type StructuredLayout = {
  schemaVersion: 'img-v3.0';
  imageMetadata: { widthPx: number; heightPx: number; calibrationMmPerPx?: number };
  rooms: Array<{
    id: string;
    polygonMm: Array<{ x: number; y: number }>;
    detectedTypeKey?: string;                // CV-detected room type from glyph / colour
    detectedAreaMm2?: number;
  }>;
  walls: Array<{ id: string; aMm: { x: number; y: number }; bMm: { x: number; y: number }; thicknessMm?: number }>;
  openings: Array<{ id: string; hostWallId: string; tAlongWall: number; widthMm?: number; kindHint?: 'door' | 'window' }>;
  ocrLabels: Array<{ text: string; bboxMm: { x: number; y: number; w: number; h: number }; confidence: number }>;
  advisories: Advisory[];                    // 'low_contrast_image', 'no_dimensions_detected', etc.
};
```
**Engine.** New CLI: `bim-ai trace --image <path> [--brief <path>] [--archetype-hint <key>] -o layout.json`. Runs SKB-04 + SKB-14 + SKB-07 (deterministic) + polygon recovery + optional OCR. Output: `(StructuredLayout, advisories)`. Registered as a tool descriptor per API-V3-01. The brief is passed through as metadata only — the deterministic CV does not interpret it; agents that read the brief do their own thing with it.
**UI.** Drop zone on canvas; "Trace as new model" enqueues a job. On completion, side-by-side preview of the source image + a SVG render of the structured layout. The user can manually correct the layout (edit walls, re-label rooms) before handing off to an agent for downstream interpretation.
**Acceptance.** (a) Drop `target-house-vis-colored.png`; trace completes in <10 s wall-clock on a single CPU; output `StructuredLayout` JSON contains ≥10 rooms and ≥40 walls. (b) Same image dropped twice → byte-identical JSON output (deterministic). (c) `bim-ai trace --image low-contrast.png` returns a layout with `low_contrast_image` advisory + degraded-but-sane polygons. (d) An external agent harness reads the layout JSON, calls CMD-V3-01 to seed the model, kernel reconstructs a valid model with green soundness pack.
**Effort.** L — 3 weeks (deterministic CV is non-trivial; OCR integration adds a week).
**Cross-theme references.** **API-V3-01 (registered tool).** **TKN-V3-01 (output JSON is round-trippable into envelope tokens).** **T6 (image-side primitives — IMG-V3-01 is the bridge from T6 to the kernel).** **JOB-V3-01 (large traces run as jobs).** **v2 SKB-04, SKB-07, SKB-14 (deterministic priors).**

## Implementation tasks

### 1. New module `app/bim_ai/img/`

Create the package:

```
app/bim_ai/img/__init__.py
app/bim_ai/img/types.py        # StructuredLayout + Advisory Pydantic models
app/bim_ai/img/pipeline.py     # trace() entry point
app/bim_ai/img/polygon.py      # Hough line → contour → room segmentation
app/bim_ai/img/ocr.py          # Tesseract wrapper (graceful fallback)
```

`types.py` — define `StructuredLayout`, `RoomRegion`, `WallSegment`, `OpeningHint`, `OcrLabel`, `Advisory` as Pydantic models matching the data model above. `schemaVersion` is a `Literal['img-v3.0']` field.

`pipeline.py` — `trace(image_path: str, *, archetype_hint: str | None = None, brief_path: str | None = None) -> StructuredLayout`. Calls SKB-04 calibrator → SKB-14 edge detection → SKB-07 colour sampler → `polygon.recover_rooms()` → `ocr.extract_labels()`. Brief is attached as `imageMetadata.briefPath` string only — not interpreted. All CV steps must be deterministic (no random seeds, no probabilistic models).

`polygon.py` — `recover_rooms(edges, calibration) -> tuple[list[RoomRegion], list[WallSegment], list[OpeningHint]]`. Implements Hough line detection → contour fitting → room segmentation. Use `opencv-python` (already in constraints; no new deps without asking).

`ocr.py` — `extract_labels(image, calibration) -> list[OcrLabel]`. Wraps `pytesseract`; on `ImportError` or Tesseract-not-installed, returns `[]` and appends `Advisory(code='tesseract_unavailable')`. Never raises.

### 2. CLI command (`packages/cli/cli.mjs`)

Register `bim-ai trace`:

```
bim-ai trace --image <path> [--brief <path>] [--archetype-hint <key>] [-o <output.json>]
```

- Calls `POST /api/v3/trace` (for large images: enqueues a job via JOB-V3-01 queue; for small images: runs inline).
- Prints `StructuredLayout` JSON to stdout (or writes to `-o` path).
- Exit code 0 on success; exit code 1 if the layout contains a fatal advisory (`no_walls_detected`).

### 3. API tool descriptor (`app/bim_ai/api/registry.py`)

Register `img_trace` tool per API-V3-01 contract: `{ name, description, inputSchema (JSON Schema for TraceImageCmd), outputSchema (JSON Schema for StructuredLayout) }`.

### 4. REST route (`app/bim_ai/routes_api.py` or new `routes_trace.py`)

`POST /api/v3/trace` — accepts `multipart/form-data` with `image` file + optional `brief` file + optional `archetypeHint` string. For images >2 MB, enqueues a `TraceJob` via the jobs queue and returns `{ jobId }`. For images ≤2 MB, runs inline and returns `StructuredLayout` JSON directly.

### 5. `TraceImageCmd` (`app/bim_ai/commands.py`)

```python
class TraceImageCmd(BaseModel):
    model_config = ConfigDict(populate_by_name=True, extra="ignore")
    type: Literal["TraceImage"] = "TraceImage"
    image_b64: str = Field(alias="imageB64")          # base64-encoded image bytes
    archetype_hint: str | None = Field(default=None, alias="archetypeHint")
    brief_text: str | None = Field(default=None, alias="briefText")
    assumptions: list = Field(default_factory=list)   # API-V3-01 assumption log
```

Add to the `Command` union. Engine handler calls `img.pipeline.trace()` and returns the `StructuredLayout` as the command result (no kernel mutation — trace is a read-only analysis tool).

### 7. UI: image drop zone (`packages/web/src/plan/ImageTraceDropZone.tsx`)

- Accepts `.png`, `.jpg`, `.jpeg`, `.pdf` via drag-and-drop or file picker.
- On drop: `POST /api/v3/trace` with `multipart/form-data`. If response is `{ jobId }`, tracks the job via JOB-V3-01; if response is `StructuredLayout`, renders immediately.
- On completion: side-by-side preview — source image on the left, SVG render of detected walls/rooms on the right.
- SVG render uses `--draft-lw-cut-major` for walls and `--color-surface-strong` for room fills at 0.15 opacity.
- No hex literals. All colours via CSS token variables.

### 8. Tests (`app/tests/test_img_v3_01.py`)

Cover:

- `trace()` on a synthetic PNG with a simple rectangle → `StructuredLayout` with ≥1 room and ≥4 walls.
- `trace()` called twice on the same file → byte-identical JSON output (determinism gate).
- Low-contrast image (flat grey PNG) → layout contains `low_contrast_image` advisory; polygons list is non-empty (degraded-but-sane).
- `ocr.extract_labels()` when Tesseract unavailable → returns `[]` + `tesseract_unavailable` advisory; does not raise.
- `TraceImageCmd` dispatched through engine → returns a dict with `schemaVersion == 'img-v3.0'`.
- `TraceImageCmd` with no `assumptions` field → still accepted (assumption log is optional for analysis-only tools).
