Never place tools here. The tracker is explicit: "Visual breath; never tools."

### 7. Slot 5 — Cmd+K button

The button must:
- render `⌘K` on macOS, `Ctrl+K` on Windows / Linux
- fire `onCommandPalette` on click
- carry `aria-keyshortcuts="Meta+K Control+K"`
- carry `data-testid="topbar-v3-cmdpalette"`
- use `--color-surface`, `--color-border`, `--color-muted-foreground`, `--radius-md`

OS detection: check `navigator.platform.toLowerCase().includes('mac')` (or `navigator.userAgent`) at render time to select the glyph string. Wrap in `useMemo` so it does not re-compute every render.

### 8. Slot 6 — Activity-stream entry

Icon button (clock-with-dot glyph). Binds `Cmd+H` globally via a `useEffect` on `document`. Fires `onActivityStream`. Carries `aria-keyshortcuts="Meta+H"` and `data-testid="topbar-v3-activity"`.

```tsx
useEffect(() => {
  const handler = (e: KeyboardEvent) => {
    if ((e.metaKey || e.ctrlKey) && e.key === 'h') {
      e.preventDefault();
      onActivityStream?.();
    }
  };
  document.addEventListener('keydown', handler);
  return () => document.removeEventListener('keydown', handler);
}, [onActivityStream]);
```

### 9. Slot 7 — Presence avatars

Show up to 4 avatars; overflow as `+N` chip. Rules:
- Each avatar: `var(--color-surface)` border, background from `presence[i].color` prop — if the caller passes a color it is always a CSS variable resolved value (never a hardcoded hex from within this component). If `presence[i].color` is undefined, fall back to `var(--color-surface-strong)`.
- Initials: first two chars of `name`, uppercased.
- Overlap: negative `margin-left: -6px` on every avatar after the first.
- `+N` chip: `var(--brand-accent)` background, `var(--brand-accent-fg)` text.
- Container: `data-testid="topbar-v3-presence"`. Hidden (`display: none`) when `presence` is empty or undefined.

### 10. Slot 8 — Profile / settings

Icon button with `data-testid="topbar-v3-account"` and `aria-label="Account"`. Shows `avatarInitials` (up to 2 chars). Background `var(--brand-accent)` at 15 % opacity via `color-mix(in srgb, var(--brand-accent) 15%, transparent)`. No hex.

Theme toggle: same pattern as existing `TopBar.tsx` `IconButton`, but use `var(--color-surface)` hover — no hardcoded hex.

### 11. Keyboard tab order

Left-to-right: logo → project name → workspace chip → Cmd+K → activity → account. No skip links needed at this density. `tabIndex={0}` on all interactive slots; `tabIndex={-1}` on the flex stretch and decorative separators.

### 12. Tooltips

Each icon button in slots 5, 6, 8 must have a `title` attribute containing the label and shortcut chip — e.g. `title="Command palette  ⌘K"`. This is the D3 antidote to A5 (tooltip-as-documentation): the shortcut is always surfaced, not hidden behind hover delay.

## Implementation rules

1. **No hex literals in chrome.** The `bim-ai/no-hex-in-chrome` ESLint rule is live. Every color value must be a CSS variable: `var(--color-*)`, `var(--disc-*)`, `var(--brand-*)`. If you need ochre, write `var(--brand-accent)`. If a caller passes a presence color as a string, that string originates from the store — the component accepts it via prop but does not define hex values itself. Violations fail CI.
2. **No Tailwind classes for color.** Use `style={}` props for all color, height, and border declarations so the values trace back to tokens unambiguously. Tailwind layout utilities (`flex`, `items-center`, `gap-*`) are fine.
3. **`--radius-canvas: 0` on the bar root.** The top bar is a rail surface. The root `div` must have `borderRadius: 'var(--radius-canvas)'`. Interior elements (buttons, chips) may use `--radius-md` or `--radius-sm`.
4. **No tools in the top bar.** The top bar is identity + meta-actions only. Do not add authoring verb buttons. This is the A3 antidote.
5. **`--text-sm` for menus, `--text-base` for project name.** These are the two type sizes specified in the tracker. `--text-2xs` may appear in tooltip shortcut chips only.
6. **16 × 16 icon size** for all icon slots. Pass `size={16}` to `Icons.*`.
7. **`--ease-paper` for any transition.** If you add a `transition` property anywhere in the component, the easing must be `var(--ease-paper)` and duration `var(--motion-slow)` (240 ms). No `ease-in-out` literals.
8. **Pure presentational component.** No Redux/Zustand imports. Data flows in as props. The component does not read from any store.
9. **File must live at `packages/web/src/workspace/chrome/TopBar.tsx`.** CHR-V3-02 will import `WorkspaceSwitcherChip` from a sibling file in the same directory.
10. **Export `TopBarV3Props` and `TopBarV3` as named exports.** `AppShell.tsx` will import the v3 surface by name under the design-system guard.
