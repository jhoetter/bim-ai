## Verbatim EDT-V3-06 tracker entry

#### EDT-V3-06 — Helper dimensions on selection (drag-the-number)

**Status.** `now`. D1 in its purest form — the dimension chip IS the input.

**Scope.** On single-element selection, render helper dimension chips at salient measurements. Each chip shows its value. Click the chip → numeric input → type → Enter commits. Tab cycles between chains. The chip IS the input field — no Inspector trip.

**Data model.** `HelperDimensionDescriptor[] = { id, valueMm, label, fromMm, toMm, onCommit: (newValueMm) => Command }`. Reuse `TempDimTarget` rendering.

**Engine.** No engine change. `onCommit` dispatches existing commands.

**Acceptance.** Select a wall — length + thickness chips render. `"5400" + Enter` commits 5.4 m wall. Tab cycles to thickness. Chips update live when geometry moves.

**Effort.** M — 1.5 weeks. Provider extension + render + per-element wiring (wall, door, window, floor, column, beam).

## Implementation tasks

### 1. `HelperDimensionDescriptor` type (`packages/core/src/index.ts`)

```ts
export type HelperDimensionDescriptor = {
  id: string;
  label: string;
  valueMm: number;
  /** Start and end points in plan mm coordinates, for drawing the dimension line */
  fromPoint: { x: number; y: number };
  toPoint: { x: number; y: number };
  /** Called when user commits a new value; returns the command to dispatch */
  onCommit: (newValueMm: number) => Record<string, unknown>;
};
```

### 2. `helperDimensions()` providers per element kind

Create `packages/web/src/plan/helperDimensions.ts` exporting a single function:

```ts
export function getHelperDimensions(
  elem: BimElem,
  model: BimModel,
): HelperDimensionDescriptor[] {
  switch (elem.kind) {
    case 'wall': return wallHelperDims(elem, model);
    case 'door': return doorHelperDims(elem, model);
    case 'window': return windowHelperDims(elem, model);
    case 'floor': return floorHelperDims(elem, model);
    case 'column': return columnHelperDims(elem, model);
    case 'beam': return beamHelperDims(elem, model);
    default: return [];
  }
}
```

Per-element helpers:

**Wall:** length (along wall axis from start to end), thickness. `onCommit` for length dispatches `{ type: 'update_wall', id, lengthMm: newValue }` (or equivalent existing command). Thickness dispatches `{ type: 'update_wall', id, thicknessMm: newValue }`.

**Door:** width. `onCommit` dispatches `{ type: 'update_door', id, widthMm: newValue }`.

**Window:** width, sill height, height. Three chips.

**Floor:** area is derived — show it read-only (chip renders but clicking opens nothing). For floors with a `widthMm` / `lengthMm`, those are editable.

**Column:** width, depth. Dispatches `{ type: 'update_column', id, widthMm, depthMm }`.

**Beam:** length (read-only from start/end points, not directly editable). Show for reference.

If an update command for a specific field doesn't exist yet, add a minimal `update_{kind}` command that accepts a partial patch object. Engine handler: look up element, apply patch via `model_copy`.

### 3. `HelperDimChip` component (`packages/web/src/plan/HelperDimChip.tsx`)

```tsx
type HelperDimChipProps = {
  descriptor: HelperDimensionDescriptor;
  planToScreen: (pt: { x: number; y: number }) => { x: number; y: number };
  onDispatch: (cmd: Record<string, unknown>) => void;
};
```

Renders as an SVG overlay on the plan canvas:

- Dimension line between `fromPoint` and `toPoint` in `var(--draft-witness)` stroke at `var(--draft-lw-witness)`
- Chip at the midpoint: `--surface-2` background, `--text-2xs` tabular-nums, `--color-foreground` text
- On click: replace chip with `<foreignObject><input type="number" /></foreignObject>`; value = current mm; Enter commits via `onCommit(parseFloat(input.value))`; Esc cancels; Tab focuses the next chip in the Tab order
- Hover: chip border highlights `var(--brand-accent)`
- **No hex literals.**

### 4. `HelperDimsLayer` component (`packages/web/src/plan/HelperDimsLayer.tsx`)

Renders as an SVG layer above the selection handles when exactly one element is selected:

```tsx
export function HelperDimsLayer({ selectedElemId, model, planToScreen, onDispatch }: { ... }) {
  const elem = model.elements[selectedElemId];
  if (!elem) return null;
  const dims = getHelperDimensions(elem, model);
  return (
    <>
      {dims.map(d => (
        <HelperDimChip
          key={d.id}
          descriptor={d}
          planToScreen={planToScreen}
          onDispatch={onDispatch}
        />
      ))}
    </>
  );
}
```

Wire into the plan canvas selection layer. The layer renders only on single-element selection (not multi-select — chips are meaningless on multiple elements).

### 5. Tab cycling

When a chip's input is focused, Tab moves focus to the next `HelperDimChip` input in DOM order. This is natural browser behaviour if the chips render in document order — verify Tab doesn't escape to the main toolbar.

### 6. Tests (`packages/web/src/plan/__tests__/helperDimensions.test.ts`)

Cover:

- `getHelperDimensions` for a wall returns descriptors with `id: 'wall-length'` and `id: 'wall-thickness'`.
- `onCommit(5400)` on the length descriptor returns the correct update command.
- `getHelperDimensions` for a door returns a descriptor for width.
- Unknown element kind returns `[]`.

No Python tests needed (frontend-only; no engine change).
