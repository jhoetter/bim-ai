## Verbatim VIE-V3-02 tracker entry (┬¦6 T4, line 1672)

**VIE-V3-02 ŌĆö Drafting-view + callout + cut-profile + view-break (R-D G18 bundle)**

**Status.** `now`.
**Scope.** **Drafting view** = `View.subKind="drafting"`, no model render; only annotation + detail components + filled regions + 2D linework. Used for typical details (eaves, head-jamb, sill) authored once and referenced from multiple sheets. **Callout** = clipped sub-view of a parent plan / section / elevation; `View.parent: viewId` + `View.clipRectInParent`. Parent shows a callout marker (rectangle + leader + circle with view number). **Cut profile** = per-view per-category override of the cut shape (layered floor ŌåÆ single line at Coarse; wall cut as outline only at small scale). Lives on `view.elementOverrides[]`. **View break** = render-time transform that hides a middle gap of a long elevation; renderer composes pre-break + post-break with a wavy cut line.

**Data model.**
```ts
{
  kind: 'view';
  subKind: 'plan' | 'section' | 'elevation' | 'drafting' | 'callout' | '3d';
  parentViewId?: string;
  clipRectInParent?: { minXY: XY; maxXY: XY };
  elementOverrides?: { categoryOrId: string; alternateRender: 'singleLine' | 'outline' | string }[];
  breaks?: { axisMM: number; widthMM: number }[];
  scale: number; detailLevel: 'coarse' | 'medium' | 'fine';
}
```

**Engine.** Drafting bypasses the projection pipeline. Callout reuses the parent's projection but clips to `clipRectInParent`. Cut profile injects per-element render alternates into the projection callback. View-break composes the rendered elevation as two viewports separated by an SVG break-line at `--draft-lw-cut-minor`.

**Effort.** XL ŌĆö 4ŌĆō5 weeks.

### 3. Engine handlers (`app/bim_ai/engine.py`)

Add dispatch cases:

**`CreateDraftingViewCmd`** ŌĆö create a `View` with `subKind='drafting'`, the given id, name, scale. Drafting views have no projection source ŌĆö they are blank canvases for 2D annotation.

**`CreateCalloutCmd`** ŌĆö create a `View` with `subKind='callout'`, `parentViewId`, `clipRectInParent`. The callout inherits the parent's projection matrix but clips to `clipRectInParent`. Also add a callout marker element to the parent view's overlay list (rectangle bounds + circle number).

**`SetElementOverrideCmd`** ŌĆö find the view by `viewId`; upsert (or append) an `ElementOverride` in `view.elementOverrides` matching `categoryOrId`. Replace if already present for that category.

**`AddViewBreakCmd`** ŌĆö append a `ViewBreak` to `view.breaks`. Sort by `axisMM` ascending.

**`RemoveViewBreakCmd`** ŌĆö remove the break whose `axisMM` matches from `view.breaks`.

Renders:

- A small circle (diameter ~18 px) at the bottom-right corner of the rectangle containing the view number at `--text-2xs`.
- A short leader line from circle to the nearest edge of the rectangle.
- Uses `--color-annotation` stroke + `--color-surface-1` circle fill.
- Double-click navigates to the callout view (call the `setActiveView` function from the view store).

### 6. View-break compositor (`packages/web/src/plan/ViewBreakCompositor.tsx`)

A component that wraps a long elevation SVG and applies breaks:

```tsx
type ViewBreakCompositorProps = {
  breaks: ViewBreak[];
  totalLengthMM: number;
  scaleDenominator: number;
  children: React.ReactNode;   // the elevation SVG content
};
```

- For each break `{ axisMM, widthMM }`: the rendered elevation is split at `axisMM`; everything between `axisMM` and `axisMM + widthMM` is hidden.
- A wavy cut-line SVG element is inserted at the break position using `stroke="var(--draft-lw-cut-minor)"`.
- The post-break portion is translated left by `widthMM ├Ś screenPixelsPerMM` to close the gap visually.

### 7. Cut-profile rendering hook (`packages/web/src/plan/useCutProfile.ts`)

```ts
export function useCutProfile(
  view: View,
  categoryOrId: string,
): 'singleLine' | 'outline' | string | null {
  return (
    view.elementOverrides?.find(o => o.categoryOrId === categoryOrId)
      ?.alternateRender ?? null
  );
}
```

The plan-canvas renderer calls this per element before projecting. When the result is `'singleLine'`, render only the single outermost cut edge. When `'outline'`, render only the outer boundary polyline.

## Implementation rules

- **Drafting views bypass projection.** `subKind='drafting'` views do not call the geometry projection pipeline. Their content is pure annotation; `resolve_visible_elements` must skip them.
- **Callout clipping is inclusive.** Elements whose bounding box intersects `clipRectInParent` are rendered; those fully outside are culled.
- **ElementOverride upsert.** There is at most one override per `categoryOrId` per view. A second `SetElementOverrideCmd` for the same category replaces the existing entry.
- **View-break widthMM must be > 0.** Reject with a violation if `widthMM <= 0`.
- **No hex literals.** CSS token variables only in TSX.
- **Additive schema.** All new fields on `View` are optional; existing views round-trip unchanged.
