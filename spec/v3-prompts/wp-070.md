## Background
CON-V3-02 defines the typed contract between T6 (concept / tracing) and T9 (refinement agent). T6 produces a `ConceptSeed` — structured layout JSON + envelope tokens + assumptions log. T9 consumes it. This WP ships the typed contract as a Python + TypeScript schema, the `CommitConceptSeedCmd` command, and the handoff endpoint.

## Implementation tasks

### 1. Python element + commands (`app/bim_ai/elements.py`, `app/bim_ai/commands.py`)

Add `ConceptSeedElem` in `app/bim_ai/elements.py`:
```python
class ConceptSeedElem(BaseModel):
    kind: Literal["concept_seed"] = "concept_seed"
    id: str
    model_id: str = Field(alias="modelId")
    source_underlay_id: Optional[str] = Field(None, alias="sourceUnderlayId")  # IMP-V3-01 ref
    envelope_tokens: list[dict] = Field(default_factory=list, alias="envelopeTokens")
    # Each token: {hostId, t, deltaMm, scaleFactor, rho}  (GBM shape)
    kernel_element_drafts: list[dict] = Field(default_factory=list, alias="kernelElementDrafts")
    # Each draft: {kind, params} — not yet committed to kernel
    assumptions_log: list[dict] = Field(default_factory=list, alias="assumptionsLog")
    # Each entry: {assumption, confidence, source}
    status: Literal["draft", "committed", "consumed"] = "draft"
    committed_at: Optional[str] = Field(None, alias="committedAt")  # ISO 8601
    schema_version: str = Field("con-v3.0", alias="schemaVersion")
```

Add commands in `app/bim_ai/commands.py`:
```python
class CommitConceptSeedCmd(BaseModel):
    """Transitions a ConceptSeedElem from draft → committed. T9 can then consume it."""
    type: Literal["commit_concept_seed"] = "commit_concept_seed"
    id: str  # ConceptSeedElem id
    envelope_tokens: Optional[list[dict]] = Field(None, alias="envelopeTokens")
    kernel_element_drafts: Optional[list[dict]] = Field(None, alias="kernelElementDrafts")
    assumptions_log: Optional[list[dict]] = Field(None, alias="assumptionsLog")

class CreateConceptSeedCmd(BaseModel):
    """Create a concept seed in draft state."""
    type: Literal["create_concept_seed"] = "create_concept_seed"
    id: str
    model_id: str = Field(alias="modelId")
    source_underlay_id: Optional[str] = Field(None, alias="sourceUnderlayId")
    envelope_tokens: list[dict] = Field(default_factory=list, alias="envelopeTokens")
    kernel_element_drafts: list[dict] = Field(default_factory=list, alias="kernelElementDrafts")
    assumptions_log: list[dict] = Field(default_factory=list, alias="assumptionsLog")

class ConsumeConceptSeedCmd(BaseModel):
    """T9 marks a seed as consumed after ingesting it."""
    type: Literal["consume_concept_seed"] = "consume_concept_seed"
    id: str
```

Add all three to `Command` union.

### 2. Engine handlers (`app/bim_ai/engine.py`)

- `CreateConceptSeed`: create element in `draft` state. Validate that `modelId` matches current model (or accept any non-empty string).
- `CommitConceptSeed`: element must exist + be in `draft` state; update to `committed` + set `committedAt` = now ISO 8601; merge any provided `envelopeTokens` / `kernelElementDrafts` / `assumptionsLog` if supplied.
- `ConsumeConceptSeed`: element must exist + be in `committed` state; update to `consumed`.

### 3. Handoff endpoint (`app/bim_ai/routes_api.py`)

Add `GET /api/v3/models/{model_id}/concept-seeds?status=committed` — returns list of committed seeds ready for T9 consumption. Filter by `status` query param.
