## Background
OUT-V3-02 builds the canvas state for a presentation deck using two primitives:
- **Frame**: rectangular crop on a presentation canvas, pointing at a `viewId` + display options
- **SavedView**: saved camera + visibility state on a view (3D/plan/sheet)

PPTX export is a structured rendering of this state — 1 Frame per slide.

### 3. PPTX export module (`app/bim_ai/exp/pptx_export.py`)

`pptx_export.py` — produce a deterministic PPTX structure:
```python
"""OUT-V3-02 — deterministic PPTX export from PresentationCanvas frames."""
from __future__ import annotations
from dataclasses import dataclass
from typing import Any
import json

@dataclass
class Slide:
    view_id: str
    caption: str | None
    position_mm: dict
    size_mm: dict
    sort_order: int

@dataclass
class PptxBundle:
    schema_version: str = "out-v3.0"
    title: str = "bim-ai presentation"
    slides: list[Slide] = None

    def to_dict(self) -> dict:
        return {
            "schemaVersion": self.schema_version,
            "title": self.title,
            "slides": [
                {
                    "viewId": s.view_id,
                    "caption": s.caption,
                    "positionMm": s.position_mm,
                    "sizeMm": s.size_mm,
                    "sortOrder": s.sort_order,
                }
                for s in sorted(self.slides or [], key=lambda s: s.sort_order)
            ],
        }

def build_pptx_bundle(canvas: dict, frames: list[dict]) -> PptxBundle:
    slides = [
        Slide(
            view_id=f["viewId"],
            caption=f.get("caption"),
            position_mm=f["positionMm"],
            size_mm=f["sizeMm"],
            sort_order=f.get("sortOrder", 0),
        )
        for f in frames
        if f.get("presentationCanvasId") == canvas["id"]
    ]
    return PptxBundle(title=canvas.get("name", "Presentation"), slides=slides)
```

Note: actual `.pptx` binary writing (via `python-pptx`) would require an additional dep. For v3, ship the structured bundle (JSON) as the PPTX contract — the binary write can be added later. The export endpoint returns the bundle JSON. Document this in the test.
