## Verbatim SHT-V3-01 tracker entry (§6 T4, line 1780)

**SHT-V3-01 — Sheet + titleblock + cartouche + window-legend (R-D G20 bundle)**

**Status.** `now`. Extends v2's partial sheet support.
**Scope.** **(a) Sheet** = printable page with view placements; **(b) Titleblock** = parametric family for project metadata + revision table + sheet number / drawn / checked / date; **(c) Cartouche** = the bottom-right titleblock cartouche per A1-landscape convention; **(d) Window legend** = a graphical legend view auto-pulling one elevation per window type, with width / height / sill height labels (distinct from a tabular schedule). Locked decisions §L: A1 landscape, titleblock strings as defined.

**Data model.**
```ts
{
  kind: 'sheet';
  id; name; number;                           // "A-101"
  size: 'A0' | 'A1' | 'A2' | 'A3'; orientation: 'landscape' | 'portrait';
  titleblockTypeId: string; revisionId?;
  viewPlacements: { viewId; minXY: XY; size: XY; scale? }[];
  metadata: { projectName, drawnBy, checkedBy, date, ... };
  brandTemplateId?;
}
{ kind: 'titleblock_type'; svgTemplate: string; tokenSlots: TokenSlot[]; }
{ kind: 'window_legend_view'; scope: 'all' | 'sheet' | 'project'; perWindowTypeRender; sortBy: 'type' | 'width' | 'count'; }
```

**Engine.** `CreateSheet` / `PlaceViewOnSheet` / `MoveViewOnSheet` / `SetSheetTitleblock`. Window-legend auto-generates: walk all windows, group by `windowTypeId`, render one elevation per group at uniform scale, label width × height × sill. Re-runs on window-type change.

**Effort.** L — 2.5 weeks.

### 4. Engine handlers (`app/bim_ai/engine.py`)

**`CreateSheetCmd`** — create a `Sheet` element with the given fields. If the `TitleblockType` referenced by `titleblockTypeId` is not in the document, auto-insert `DEFAULT_TITLEBLOCK_TYPE` first.

**`PlaceViewOnSheetCmd`** — find the sheet; append a `ViewPlacement` to `sheet.viewPlacements`. If a placement for the same `viewId` already exists on this sheet, replace it (a view can appear on multiple sheets but only once per sheet).

**`MoveViewOnSheetCmd`** — find the placement by `viewId` on the sheet; update `minXY`.

**`RemoveViewFromSheetCmd`** — remove the placement with the matching `viewId`.

**`SetSheetTitleblockCmd`** — update `sheet.titleblockTypeId`.

**`UpdateSheetMetadataCmd`** — merge `metadata` dict into `sheet.metadata` (patch semantics — only keys present in the command body are updated).

**`CreateWindowLegendViewCmd`** — create a `WindowLegendView` element. Window legend generation is deferred to render time; no geometry is computed in the command handler.

### 5. Window legend render helper (`app/bim_ai/sheets.py`)

New file:

```python
from __future__ import annotations
from bim_ai.document import Document, WindowLegendView

def resolve_window_legend(doc: Document, legend: WindowLegendView) -> list[dict]:
    """Return one entry per unique window type: {typeId, label, widthMm, heightMm, sillMm}.

    Walks doc.elements, groups windows by windowTypeId, returns sorted per legend.sortBy.
    """
    windows = [e for e in doc.elements.values() if getattr(e, 'kind', None) == 'window']
    by_type: dict[str, list] = {}
    for w in windows:
        tid = getattr(w, 'window_type_id', None) or 'unknown'
        by_type.setdefault(tid, []).append(w)

    entries = []
    for tid, instances in by_type.items():
        sample = instances[0]
        entries.append({
            "typeId": tid,
            "label": getattr(sample, 'type_mark', tid),
            "widthMm": getattr(sample, 'width_mm', 0),
            "heightMm": getattr(sample, 'height_mm', 0),
            "sillMm": getattr(sample, 'sill_height_mm', 0),
            "count": len(instances),
        })

    sort_key = {"type": "label", "width": "widthMm", "count": "count"}.get(legend.sort_by, "label")
    entries.sort(key=lambda e: e[sort_key])
    return entries
```

## Implementation rules

- **Default titleblock is auto-inserted.** If `CreateSheetCmd` references a `titleblockTypeId` not in the document, auto-insert `DEFAULT_TITLEBLOCK_TYPE`. Workers must not require the UI to pre-create a titleblock type before creating a sheet.
- **A view can appear on multiple sheets.** `PlaceViewOnSheet` does not prevent the same `viewId` from appearing on different sheets. It only prevents duplicates on the same sheet.
- **`UpdateSheetMetadata` is patch (not replace).** Only keys present in the command's `metadata` dict overwrite the sheet's metadata. Absent keys are preserved.
- **Window legend render is deferred.** `CreateWindowLegendViewCmd` stores the configuration; actual legend generation runs at render time via `resolve_window_legend`. No geometry is pre-baked.
- **No hex literals.** CSS token variables only in TSX.
- **Additive schema.** `Sheet`, `TitleblockType`, `WindowLegendView` are new element kinds; existing documents round-trip unchanged.
