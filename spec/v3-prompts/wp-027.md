## Verbatim VER-V3-01 tracker entry (§6 T3, line 1366)

**VER-V3-01 — Activity stream as time-machine (Linear-grade)**

**Status.** `now`.
**Scope.** Every commit, comment, markup, snapshot, option-set event renders as a row in the activity stream. Rows are clickable to time-travel. Read-write via yjs commit log. Visual quality: Linear-grade — dense, scannable, avatars + verb + noun + timestamp. Filterable by author / type / date. Row hover preview-renders the change (canvas dims, change highlights at `--cat-edit` color).

**Data model.**
```ts
type ActivityRow = {
  id: string;
  modelId: string;
  authorId: string;
  kind:
    | 'commit'
    | 'comment_created' | 'comment_resolved'
    | 'markup_created' | 'markup_resolved'
    | 'milestone_created'
    | 'option_set_lifecycle'
    | 'collab_join' | 'collab_leave';
  payload: Record<string, unknown>;
  ts: number;
  parentSnapshotId?: string;
  resultSnapshotId?: string;
};
```

**Engine.** Server materialises an `activity_rows` table per model, indexed on `modelId + ts`. Time-travel: click a row → canvas reads `parentSnapshotId` in read-only with a "viewing version N" banner; "Restore" creates a fresh commit that forward-copies the historical state.

**Effort.** L — 2.5 weeks.

### 1. Database table (`app/bim_ai/tables.py`)

Add an `ActivityRow` SQLAlchemy model:

```python
class ActivityRowRecord(Base):
    __tablename__ = "activity_rows"
    id: Mapped[str] = mapped_column(String, primary_key=True)
    model_id: Mapped[str] = mapped_column(String, ForeignKey("models.id", ondelete="CASCADE"), index=True)
```

```python
from __future__ import annotations
from typing import Any, Literal
from pydantic import BaseModel, ConfigDict, Field

ActivityKind = Literal[
```

### 4. REST routes (`routes_api.py`)

Add two routes:

- `GET /models/{model_id}/activity` — return paginated activity rows, most recent first. Query params: `?limit=50&before=<ts>&kind=<kind>&authorId=<id>`.
- `POST /models/{model_id}/activity/{row_id}/restore` — time-travel + restore. Fetches the snapshot at `parentSnapshotId`, re-applies it as a new commit forward-copying state.

## Implementation rules

- **Emit on every successful commit.** The `emit_activity_row` call must be inside the `if result.applied` branch in `apply_bundle_route`, after `await session.commit()`.
- **Restore is additive.** Restore never deletes rows. It creates a new commit row at the top. Time-travel UI is read-only; only "Restore" commits state.
- **Pagination via cursor, not offset.** Use `before=<ts>` cursor for stable pagination.
- **No hex literals.** CSS token variables only in TSX.
- **Additive schema.** `activity_rows` table is new; no migrations to existing tables.
