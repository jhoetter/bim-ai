## Implementation tasks

### 1. Site wall binding — extend `WallElem` / `UpdateWallCmd`

In `app/bim_ai/elements.py`, add optional `site_host_id: Optional[str] = Field(None, alias="siteHostId")` to `WallElem`.

In `app/bim_ai/commands.py`, add `site_host_id: Optional[str] = Field(None, alias="siteHostId")` to both `CreateWallCmd` and `UpdateWallCmd`.

Engine rule: when `site_host_id` is set, the wall is a site wall. At render/export time the engine attaches a note that base elevation per-segment follows the surface (actual 3D slope rendering is a visual layer; the data contract is the binding). If `site_host_id` references a nonexistent toposolid, return 400.

In `packages/core/src/index.ts`, add `siteHostId?: string` to `WallElem`, `CreateWallCmd`, `UpdateWallCmd`.

### 2. Graded region element + commands

Add `GradedRegionElem` in `app/bim_ai/elements.py`:
```python
class GradedRegionElem(BaseModel):
    kind: Literal["graded_region"] = "graded_region"
    id: str
    host_toposolid_id: str = Field(alias="hostToposolidId")
    boundary_mm: list[dict] = Field(alias="boundaryMm")  # [{xMm, yMm}]
    target_mode: Literal["flat", "slope"] = Field("flat", alias="targetMode")
    target_z_mm: Optional[float] = Field(None, alias="targetZMm")
    slope_axis_deg: Optional[float] = Field(None, alias="slopeAxisDeg")
    slope_deg_percent: Optional[float] = Field(None, alias="slopeDegPercent")
```

Add commands: `CreateGradedRegionCmd`, `UpdateGradedRegionCmd`, `DeleteGradedRegionCmd` — following the same pattern as `CreateToposolidSubdivisionCmd` in elements.py.

Add to `Command` union in `commands.py`.

Engine handlers:
- Create: host toposolid must exist; for `flat` mode `targetZMm` is required; for `slope` mode both `slopeAxisDeg` and `slopeDegPercent` required.
- Update: element must exist; partial updates OK.
- Delete: remove.
