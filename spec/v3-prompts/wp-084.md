## Implementation tasks

### 1. Remove the "Tool:" prefix and add verb-prefixed tool labels

Open `packages/web/src/workspace/StatusBar.tsx`. Find the `ToolCluster` component (around line 314). It currently renders something like:
```tsx
{t('statusbar.toolLabel')}{' '}
<span className="font-medium text-foreground">{toolLabel ?? '—'}</span>
```

Replace with a verb-prefix approach. Add a verb map near the top of `StatusBar.tsx`:

```typescript
const TOOL_VERB: Record<string, string> = {
  wall: 'Drawing wall',
  door: 'Placing door',
  window: 'Placing window',
  floor: 'Sketching floor',
  roof: 'Sketching roof',
  stair: 'Sketching stair',
  railing: 'Placing railing',
  column: 'Placing column',
  beam: 'Placing beam',
  room: 'Bounding room',
  dimension: 'Annotating',
  section: 'Placing section',
  elevation: 'Placing elevation',
  callout: 'Placing callout',
  detail_region: 'Framing detail',
  annotation: 'Annotating',
  grid: 'Drawing grid',
  level: 'Setting level',
};

function toolVerb(toolLabel: string | null | undefined): string {
  if (!toolLabel) return '—';
  const key = toolLabel.toLowerCase().replace(/\s+/g, '_');
  return TOOL_VERB[key] ?? toolLabel;
}
```

Update `ToolCluster` to just render `toolVerb(toolLabel)` — no prefix label. The i18n key `statusbar.toolLabel` can be removed from the component (check if it's used elsewhere before deleting the translation key).

### 2. Snap-mode icons instead of text chips

Find `SnapCluster` in `StatusBar.tsx`. Currently it renders text chips for each snap mode (something like "ENE", "MID", "INT" abbreviations). Replace with icon glyphs from `@bim-ai/icons`.

Read `packages/icons/src/` to find available snap-related icons. Look for `EndpointIcon`, `MidpointIcon`, `IntersectionIcon`, `PerpendicularIcon`, `GridIcon`, `NearestIcon`. If those exact names don't exist, find the closest available icon.

For each snap mode chip, replace the text label with the icon component at `size={14}` (or the `ICON_SIZE.chrome` constant used elsewhere in StatusBar). Keep the `title` attribute for tooltip accessibility.

If snap icons genuinely don't exist in the icon set, use a single generic snap glyph (`Icons.snap` or similar) for all modes rather than text — a consistent icon is better than text abbreviations. Note any missing icons in the final report for a follow-up.

### 3. Coordinate readout in metres

Find `CoordCluster` in `StatusBar.tsx`. It currently formats `cursorMm` as raw integer millimetres. Update to show metres with 3 decimal places:

```typescript
function formatCoord(mm: number | undefined): string {
  if (mm === undefined || mm === null) return '—';
  return `${(mm / 1000).toFixed(3)} m`;
}
```

Replace the `xMm`/`yMm` shorthand labels with just `x` and `y` (or omit the labels entirely if the readout is clear enough from context):

```tsx
<span>x {formatCoord(cursorMm?.xMm)}</span>
<span>y {formatCoord(cursorMm?.yMm)}</span>
```

Add any new keys needed (e.g. `statusbar.x`, `statusbar.y` coordinate labels if they use translation keys). Keep other translation keys (snap, coords, level) intact.

Do not delete tests — only update expectations to match the new format.
