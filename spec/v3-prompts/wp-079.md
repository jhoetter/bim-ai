## Verbatim tracker entry (§6 T10 VIS-V3-02)

**Scope.** `--font-sans: 'Inter', ui-sans-serif, ...` is declared in `tokens-default.css` but Inter is never loaded — no `@import`, no `@font-face`, no self-hosted copy. The browser falls back to `system-ui`. Add the import in `brand-layer.css` so it loads for every design system.

## Implementation tasks

### 1. Add Inter import to `brand-layer.css`

Open `packages/design-tokens/src/brand-layer.css`. Add a `@import` as the very first line of the file (before the comment block and `:root` block):

```css
/* Inter — loaded for all design systems via brand-layer.css.
 * Bunny Fonts: GDPR-compliant EU CDN, zero Google tracking. */
@import url('https://fonts.bunny.net/css?family=inter:400,500,600&display=swap');
```

Weights loaded:
- 400 — body text, labels
- 500 — medium weight (toolbar labels, panel headings)
- 600 — section headings, emphasis (NOT 700/800 — brand display headline uses `--brand-typeface` override)

Do NOT use Google Fonts (`fonts.googleapis.com`) — use Bunny Fonts (`fonts.bunny.net`) which is GDPR-compliant and appropriate for EU customers.

### 2. Confirm `--font-sans` references Inter

Open `packages/design-tokens/src/tokens-default.css`. Confirm the `--font-sans` declaration looks like:
```css
--font-sans: 'Inter', ui-sans-serif, system-ui, -apple-system, ...;
```
No change needed — just confirm it's there and that the loaded font name matches.

No TypeScript errors expected — this is a CSS-only change.

## Implementation rules

- **Use Bunny Fonts, not Google Fonts.** EU customers care about GDPR; using Google would be a support issue.
- **No scope creep.** Do not add `@font-face` self-hosting in this WP — that's a separate follow-up if needed. The Bunny CDN import is sufficient.
- **@import must be the first line.** CSS `@import` rules must appear before all other rules (including comments that aren't at the very top) to be valid. Place it as the first content in the file.
