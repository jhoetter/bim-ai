## Implementation tasks

### 1. Make v3 the default design system (`packages/web/vite.config.ts`)

Open `packages/web/vite.config.ts`. The `resolveDesignSystemId` function currently returns `'default'` in two places:

```typescript
// Line ~17 — the nullish chain fallback:
env.VITE_DESIGN_SYSTEM ??
env.DESIGN_SYSTEM ??
process.env.VITE_DESIGN_SYSTEM ??
process.env.DESIGN_SYSTEM ??
'default'   // ← change to 'v3'

// Line ~25 — the warn-and-return fallback:
console.warn(`[bim-ai] Unknown VITE_DESIGN_SYSTEM="${raw}". Falling back to default.`);
return 'default';  // ← change to return 'v3'
```

Change both `'default'` strings to `'v3'`. Result: any environment without an explicit `VITE_DESIGN_SYSTEM` env var now loads the v3 warm ochre token system. Environments that explicitly set `VITE_DESIGN_SYSTEM=default` (or `=playful`, `=conservative`) continue to work — the env var still takes priority.

### 2. Add warm dark chrome tokens (`packages/design-tokens/src/tokens-v3.css`)

Open `packages/design-tokens/src/tokens-v3.css`. The existing `:root[data-theme='dark'], :root.dark` block only overrides `--color-accent` and `--color-drift`. Extend it to add warm background, surface, and border tokens:

```css
:root[data-theme='dark'],
:root.dark {
  /* Chrome accent — warm ochre dark variant */
  --color-accent: #d9924a;
  --color-accent-foreground: hsl(220 18% 10%);
  /* Fix accent-soft blend target to match warm background */
  --color-accent-soft: color-mix(in srgb, #d9924a 18%, hsl(25 8% 9%));
  --color-ring: color-mix(in srgb, #d9924a 38%, transparent);

  /* Drift severity — dark variant */
  --color-drift: #d2a256;

  /* VIS-V3-01 — warm dark chrome background + surfaces */
  /* Replaces the cold blue-grey from tokens-dark.css */
  --color-background: hsl(25 8% 9%);
  --color-surface-1: hsl(25 6% 12%);
  --color-surface-2: hsl(25 5% 15%);
  --color-surface-3: hsl(25 5% 18%);
  --color-border: hsl(25 8% 22%);
  --color-border-strong: hsl(25 8% 28%);
}
```

Important: read `tokens-dark.css` first to confirm the exact token names used for surfaces and borders. If `tokens-dark.css` uses different names (e.g. `--color-card`, `--color-popover`), add warm overrides for those names too.

Do NOT add any Layer-A tokens (drafting, line-weight, motion) here — those must never be themed per §3.
