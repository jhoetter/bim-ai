## Verbatim KRN-V3-04 tracker entry (§6 T1, line 614)

**KRN-V3-04 — Design Options**

_Source: R-D §3 G5. Severity High. Gates B2 Sketch-to-Deck **and** T9 — without options, every agent edit destructively overwrites the host model._

**Status.** `now`.
**Scope.** Multiple alternatives (Option A vs B kitchen) coexisting in the same model. A `DesignOptionSet` is project-level with N `DesignOption` children; one is `primary`. Elements taggable with `optionId`; default null = "main" (always rendered). Views can lock to a specific option. Activity-stream / undo aware. The agent in T9 uses Design Options as its scratch surface for proposed edits.

**Data model.**
```ts
type DesignOptionSet = {
  id: string;
  name: string;       // e.g. "Kitchen variant"
  options: Array<{
    id: string;
    name: string;     // e.g. "A — galley", "B — island"
    isPrimary: boolean;
  }>;
};

// Element-level
optionSetId?: string;    // null = main, present in every option
optionId?: string;       // null = main; else the specific option

// View-level
optionLocks: Record<string /* optionSetId */, string /* optionId */>;
```
**Engine.** Commands: `CreateOptionSet`, `AddOption`, `RemoveOption`, `RenameOption`, `SetPrimaryOption`, `AssignElementToOption`, `SetViewOptionLock`. Snapshot resolution: when a view computes its element set, for each option-set it picks the locked option (or the primary if unlocked); elements with no `optionId` are always included.

**Effort.** L — 2.5 weeks.

### 4. Engine — dispatch (`engine.py`)

Add handlers for all six commands in `apply_inplace`:

- `CreateOptionSetCmd`: validate id not duplicate, append to `doc.design_option_sets`.
- `AddOptionCmd`: validate set exists, id not duplicate within set, if `isPrimary=True` clear `isPrimary` on all existing options in the set first.
- `RemoveOptionCmd`: validate set + option exist; reject if this is the only option in the set with a `ValueError`; move elements in that option back to main (clear `option_id`/`option_set_id`).
- `SetPrimaryOptionCmd`: validate set + option exist; clear `isPrimary` on all others, set on the target.
- `AssignElementToOptionCmd`: validate element exists; validate optionSetId/optionId both non-null or both null; if non-null, validate the set + option exist.
- `SetViewOptionLockCmd`: validate view exists; if `optionId` is None, remove key from `option_locks`; else validate set + option exist, set the lock.

### 5. Snapshot resolution helper (`engine.py`)

Add a pure function:

```python
def resolve_visible_elements(
    doc: "Document",
    option_locks: dict[str, str],
) -> list[str]:
    """Return the element IDs visible under the given option locks.

    An element is visible if:
    - It has no optionSetId (belongs to main).
    - Its optionSetId is locked in option_locks and its optionId matches the locked value.
    - Its optionSetId is NOT locked in option_locks but it is the primary option for that set.
    """
    visible = []
    for eid, elem in doc.elements.items():
        set_id = getattr(elem, "option_set_id", None)
        opt_id = getattr(elem, "option_id", None)
        if set_id is None:
            visible.append(eid)
            continue
        locked = option_locks.get(set_id)
        if locked is not None:
            if opt_id == locked:
                visible.append(eid)
        else:
            # no lock — use primary
            the_set = next((s for s in doc.design_option_sets if s.id == set_id), None)
            if the_set is not None:
                primary = next((o for o in the_set.options if o.is_primary), None)
                if primary is not None and opt_id == primary.id:
                    visible.append(eid)
    return visible
```
