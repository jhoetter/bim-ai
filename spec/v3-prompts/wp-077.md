## Background
COL-V3-06 ships offline-tolerant authoring: user loses connection → continues editing locally → on reconnect, ops sync + CRDT merge converges. No "you're offline" modal that blocks work.

This WP ships:
1. A network-status store slice (online/offline state tracking)
2. An offline indicator on the participant strip (ochre dot)
3. A command queue that buffers commands while offline and drains on reconnect
4. Activity-stream "synced from offline" badge type
5. Tests for the queue drain logic

No actual CRDT library integration required in this WP — the CRDT merge is provided by yjs (COL-V3-01). This WP ships the **UI contract** and the **queue draining pattern**. The actual IndexedDB persistence (y-indexeddb) is wired once yjs is confirmed working in the deployment (post-v3 integration step).

## Implementation tasks

### 1. Network status store (`packages/web/src/offlineStore.ts`)

```typescript
import { create } from 'zustand';

type OfflineStore = {
  isOnline: boolean;
  pendingCommandCount: number;
  lastSyncedAt: string | null;
  offlineQueuedAt: string | null;  // when we went offline
  setOnline: (online: boolean) => void;
  incrementPendingCount: () => void;
  clearPendingCount: () => void;
  setLastSyncedAt: (ts: string) => void;
};

export const useOfflineStore = create<OfflineStore>((set) => ({
  isOnline: navigator.onLine,
  pendingCommandCount: 0,
  lastSyncedAt: null,
  offlineQueuedAt: null,
  setOnline: (isOnline) => set((s) => ({
    isOnline,
    offlineQueuedAt: !isOnline && s.isOnline ? new Date().toISOString() : s.offlineQueuedAt,
  })),
  incrementPendingCount: () => set((s) => ({ pendingCommandCount: s.pendingCommandCount + 1 })),
  clearPendingCount: () => set({ pendingCommandCount: 0, lastSyncedAt: new Date().toISOString() }),
  setLastSyncedAt: (ts) => set({ lastSyncedAt: ts }),
}));

// Wire to browser online/offline events
if (typeof window !== 'undefined') {
  window.addEventListener('online', () => useOfflineStore.getState().setOnline(true));
  window.addEventListener('offline', () => useOfflineStore.getState().setOnline(false));
}
```

### 2. Offline command queue (`packages/web/src/commandQueue.ts`)

```typescript
/**
 * COL-V3-06 — offline command queue.
 * Buffers commands when offline; drains to the API on reconnect.
 */
import type { Command } from '@bim-ai/core';

type QueueEntry = {
  command: Command;
  modelId: string;
  enqueuedAt: string;
};

let _queue: QueueEntry[] = [];

export function enqueueCommand(command: Command, modelId: string): void {
  _queue.push({ command, modelId, enqueuedAt: new Date().toISOString() });
}

export function getQueueLength(): number {
  return _queue.length;
}

export async function drainQueue(
  applyBundle: (modelId: string, commands: Command[]) => Promise<void>
): Promise<{ drained: number; errors: number }> {
  if (_queue.length === 0) return { drained: 0, errors: 0 };
  
  // Group by modelId for efficient bundle apply
  const byModel = new Map<string, Command[]>();
  for (const entry of _queue) {
    if (!byModel.has(entry.modelId)) byModel.set(entry.modelId, []);
    byModel.get(entry.modelId)!.push(entry.command);
  }
  
  let drained = 0, errors = 0;
  for (const [modelId, commands] of byModel) {
    try {
      await applyBundle(modelId, commands);
      drained += commands.length;
    } catch {
      errors += commands.length;
    }
  }
  
  if (errors === 0) _queue = [];
  return { drained, errors };
}

export function clearQueue(): void { _queue = []; }
```

### 5. Tests (`packages/web/src/__tests__/offlineQueue.test.ts`)

Write at least 10 tests:
- `enqueueCommand` adds to queue
- `getQueueLength` returns correct count
- `drainQueue` calls `applyBundle` with correct commands grouped by modelId
- `drainQueue` clears queue on success
- `drainQueue` preserves queue on error
- `drainQueue` returns correct `{drained, errors}` counts

- `setOnline(false)` sets `offlineQueuedAt`
- `setOnline(true)` when already online does not change `offlineQueuedAt`
- `incrementPendingCount` increments
- `clearPendingCount` resets to 0 and sets `lastSyncedAt`
