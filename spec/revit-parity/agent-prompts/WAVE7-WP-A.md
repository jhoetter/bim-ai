### B — WallTypeLayerEditor component

Create `packages/web/src/workspace/families/WallTypeLayerEditor.tsx`:

```tsx
interface WallTypeLayerEditorProps {
  typeElement: Extract<Element, { kind: 'wall_type' | 'floor_type' | 'roof_type' }>;
  onUpdate: (patch: { name?: string; layers?: WallTypeLayer[]; basisLine?: string }) => void;
}
export function WallTypeLayerEditor({ typeElement, onUpdate }: WallTypeLayerEditorProps): JSX.Element
```

The editor:
- `data-testid="wall-type-layer-editor"` on container
- Type name input at top (`data-testid="wall-type-name-input"`) — editable, calls `onUpdate({ name })`
- Table of layers (`data-testid="wall-type-layers-table"`):
  - Each row has `data-testid="wall-type-layer-row-{i}"`
  - Thickness input (mm) `data-testid="wall-type-layer-thickness-{i}"`
  - Function select `data-testid="wall-type-layer-function-{i}"` — structure / insulation / finish / membrane / air
  - Material key input `data-testid="wall-type-layer-material-{i}"` (text — e.g. `'brick_facing'`)
  - Delete button `data-testid="wall-type-layer-delete-{i}"` → removes layer at index i
  - Up/Down buttons `data-testid="wall-type-layer-up-{i}"` / `data-testid="wall-type-layer-down-{i}"` to reorder
- "Add Layer" button `data-testid="wall-type-add-layer"` → appends `{ thicknessMm: 100, function: 'structure', materialKey: null }`
- All mutations call `onUpdate({ layers: newLayers })` — caller dispatches `update_wall_type`
