## Implementation tasks

### 1. Recalibrate hemisphere light (`packages/web/src/viewport/materials.ts`)

Find the `resolveLighting()` function. Update the `hemi` block:

```typescript
hemi: {
  intensity: 0.45,
  skyColor: '#e0d8cc',    // was '#d3e2ff' (cold steel-blue → warm stone)
  groundColor: '#c8c0b0', // was '#d8d3c4' (cooler sand → warm sand)
},
```

The sky colour is the single biggest source of the cold tint that washes over all 3D materials. `#d3e2ff` is a cool lavender-blue that bleeds into everything via the env map; `#e0d8cc` is warm stone that matches the architectural tone of the `tokens-v3.css` palette.

### 2. Reduce env map intensity (`packages/web/src/Viewport.tsx`)

Search for `envMapIntensity` in `Viewport.tsx`. There is a line that sets it per material:

```typescript
envMapIntensity: csgIsWhite ? 0.08 : 1.0,
```

Change `1.0` to `0.65`:

```typescript
envMapIntensity: csgIsWhite ? 0.08 : 0.65,
```

At `1.0`, the sky env map dominates material colour; at `0.65`, it adds realistic reflective contribution without bleaching or tinting the base material colour. The CSG white special case (`0.08`) remains unchanged.

### 3. Recalibrate category colours (`packages/design-tokens/src/tokens-drafting.css`)

Open `tokens-drafting.css`. There are two `:root` blocks: one for light mode (without media query or with a light selector) and one for dark mode. Update the following tokens:

**Dark mode block (search for `--cat-wall: #7a7d83`):**
```css
--cat-wall: #a09a92;     /* warm plaster grey — was #7a7d83 (cold grey) */
--cat-roof: #7a5c4a;     /* clay slate — was #944f4f (vivid red-terracotta) */
--cat-railing: #5a5650;  /* warm dark grey — was #43464d (cold) */
```

**Light mode block (search for `--cat-wall: #969aa1`):**
```css
--cat-wall: #b0aa9e;     /* warm plaster — was #969aa1 (cold grey) */
--cat-roof: #6b4f3e;     /* dark clay — was #7a4949 (red-brown) */
--cat-railing: #706b64;  /* warm grey — was #54585f (cold) */
```

Leave `--cat-floor`, `--cat-door`, `--cat-window`, `--cat-stair` unchanged — they are already well-calibrated for the warm-sky world.

## Implementation rules

- **Targeted changes only.** Touch only `materials.ts` (hemi colours), `Viewport.tsx` (envMapIntensity line), and `tokens-drafting.css` (the three `--cat-*` tokens). Do not refactor the materials system or add new token infrastructure in this WP.
- **No hex literals in components.** `Viewport.tsx` already has the exemption comment `/* eslint-disable bim-ai/no-hex-in-chrome -- pre-v3 hex literals */`.
