## Verbatim OPT-V3-01 tracker entry (§6 T9, line 3202)

**OPT-V3-01 — Design Options as agent scratch surface (non-destruction invariant)**

**Status.** `now`.
**Scope.** T9's contribution to KRN-V3-04: the kernel-level non-destruction invariant. **CMD-V3-01 cannot directly commit to `main`** unless the caller explicitly sets `targetOptionId: 'main'` AND has the `commit-to-main` capability. Every bundle without `targetOptionId` lands as a new `DesignOption`. Humans hit "Promote to main" via deliberate click; agents cannot bypass that ceremony. Provenance recorded in the option's `provenance` field.

**Data model.** Adds `provenance` field to `DesignOption`:
```ts
provenance?: {
  submitter: 'agent' | 'human' | 'ci';
  bundleId: string;
  createdAt: number;
};
```

**Engine.** Wrapper inside CMD-V3-01: every successful apply targets a fresh option unless bundle explicitly opts into `main` AND the caller has `commit-to-main` capability. Rejected with `direct_main_commit_forbidden` otherwise.

**Effort.** M — 1.5 weeks.

### 3. Non-destruction gate in `apply_bundle` (`app/bim_ai/cmd/apply_bundle.py`)

**Current behaviour:** apply_bundle commits all commands directly to the document.

**New behaviour:** check `bundle.target_option_id`:

```python
def _resolve_bundle_target(
    doc: Document,
    bundle: CommandBundle,
    submitter: str = "human",
) -> tuple[str | None, str | None]:
    """Return (option_set_id, option_id) for the bundle's target.

    Returns (None, None) if targeting main (only allowed when targetOptionId=='main').
    Returns (set_id, new_option_id) if auto-routing to a new option.
    Raises ValueError('direct_main_commit_forbidden') if targetOptionId=='main'
    but submitter is 'agent'.
    """
    if bundle.target_option_id == "main":
        if submitter == "agent":
            raise ValueError("direct_main_commit_forbidden")
        return (None, None)  # commit to main
    # Auto-route: create a new DesignOption in a default "Agent proposals" set
    ...
```

When `target_option_id` is not `'main'` and not `None`, route to the specified option. When `None`, create a fresh option in a "Agent proposals" option set (auto-create the set if it doesn't exist).

### 4. Advisory class in `apply_bundle`

If `direct_main_commit_forbidden` is raised, surface it as a bundle violation with `advisoryClass='direct_main_commit_forbidden'` (this class is already in the blocking set in `routes_api.py`):

## Implementation rules

- **The invariant is in `apply_bundle`, not in the route.** The route accepts any bundle; `apply_bundle` enforces the non-destruction gate. This way the CLI, REST, and any other caller all obey the same rule.
- **"Agent proposals" set is auto-created.** Workers must not require the UI to pre-create an option set before submitting a bundle.
- **`direct_main_commit_forbidden` is already in the blocking advisory class set** in `routes_api.py`. No change needed there.
- **Additive schema.** `provenance` on `DesignOption` is optional; existing options without it round-trip unchanged.
- **No hex literals in TSX.**
