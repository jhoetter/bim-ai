## Implementation tasks

### 1. Find the AppShell composition site

Open `packages/web/src/workspace/Workspace.tsx`. Find where `<AppShell>` is rendered. It will have props like:
```tsx
<AppShell
  topBar={...}
  leftRail={...}
  canvas={...}
  rightRail={...}
  statusBar={...}
  defaultLeftCollapsed={false}    // ← change to true
  defaultRightCollapsed={false}   // ← drive from selection state
/>
```

### 2. Default left rail to collapsed

Change `defaultLeftCollapsed={false}` to `defaultLeftCollapsed={true}`.

This means on page load the left rail shows only the 36 px icon strip (the collapsed state). Users press `[` to expand. The CHR-V3-07 project browser (WP-075) ships the collapsed icon-strip content — it should already be wired into `leftRailCollapsed` prop.

### 3. Drive right rail from selection state

Find the existing selection store. It will be something like `useSelectionStore`, `useElementSelection`, or a property on the model store. Specifically look for a hook or selector that returns the currently selected element IDs or the selection count.

```typescript
const hasSelection = useSelectionStore((s) => s.selectedIds.length > 0);
// OR
const hasSelection = usePlanStore((s) => s.selectedElementIds.length > 0);
```

Wire this to the right-rail collapsed state:

```tsx
<AppShell
  defaultLeftCollapsed={true}
  rightCollapsed={!hasSelection}
  ...
/>
```

Note: `AppShell` has both `defaultRightCollapsed` (initial state, uncontrolled) and a `rightCollapsed` controlled prop. Use the controlled prop (`rightCollapsed={!hasSelection}`) for selection-driven behaviour. Read `AppShell.tsx` to confirm the prop name and controlled vs. uncontrolled API.

### 4. Ensure the right rail slides in with motion

The existing AppShell CSS should already handle the rail width transition. Check the CSS grid transition in `AppShell.tsx`. If the right rail transitions are missing or instant, add:

```css
/* In AppShell's grid-template-columns: use CSS transition */
transition: grid-template-columns 200ms var(--ease-paper);
```

If the grid transition approach isn't feasible (CSS grid transitions have browser support nuances), the fallback is `width` transition on the right rail element itself. The `--ease-paper` easing is the spec'd easing for commit/settle moments.

### 5. Update existing AppShell tests

Open `packages/web/src/workspace/AppShell.test.tsx`. Some tests likely assert `data-left-collapsed="false"` as the default. Update them to `data-left-collapsed="true"` to reflect the new default. Do not delete tests — just update expectations.
