## Verbatim DSC-V3-02 tracker entry

#### DSC-V3-02 — Discipline tags on views

**Status.** `now`.
**Scope.** Every view gains a `defaultLens: LensMode`. Values: `'show_arch'`, `'show_struct'`, `'show_mep'`, `'show_all'`. Default at view-create inherits from active workspace's `defaultLens`. The lens is a view modifier — does not mutate the model; changes which elements are foreground vs ghost.
**Data model.**
```ts
type LensMode = 'show_arch' | 'show_struct' | 'show_mep' | 'show_all';
defaultLens: LensMode;  // view-level; default = 'show_all'
```
**Engine.** Snapshot resolution: element matches lens (or lens is `show_all`) → foreground; otherwise ghost at 25 % opacity. `'show_all'` puts everything full opacity. Hidden categories override the lens (separate axis).
**Acceptance.** Create a view → `defaultLens = 'show_struct'`. Arch walls render 25 % ghost; structural elements stay foreground. Switching workspace doesn't mutate views.
**Effort.** M — 1.5 weeks.

### 1. `LensMode` type + `SetViewLensCmd` (`app/bim_ai/commands.py`)

```python
from typing import Literal
LensMode = Literal["show_arch", "show_struct", "show_mep", "show_all"]

class SetViewLensCmd(BaseModel):
    model_config = ConfigDict(populate_by_name=True, extra="ignore")
    type: Literal["set_view_lens"] = "set_view_lens"
    view_id: str = Field(alias="viewId")
    lens: LensMode
```

Add `SetViewLensCmd` to the `Command` union.

### 2. `defaultLens` on view elements (`app/bim_ai/elements.py`)

All view element classes (FloorPlanView, SectionView, ElevationView, DraftingView, CalloutView, View3dElem — whatever exists) gain:


If views are stored in a generic dict rather than typed classes, ensure `defaultLens` defaults to `"show_all"` in the snapshot serialisation.

### 3. Engine handler (`app/bim_ai/engine.py`)

In `apply_inplace`:

```python
case SetViewLensCmd():
    view = model.elements.get(cmd.view_id)
    if view is None:
        raise ValueError(f"View not found: {cmd.view_id}")
    model.elements[cmd.view_id] = view.model_copy(update={"default_lens": cmd.lens})
```

Also implement a helper used by the rendering layer:

```python
def element_passes_lens(elem_discipline: str | None, lens: str) -> bool:
    """Return True if the element should be foreground under the given lens."""
    if lens == "show_all":
        return True
    mapping = {"show_arch": "arch", "show_struct": "struct", "show_mep": "mep"}
    return elem_discipline == mapping.get(lens)
```

### 6. Ghost/foreground rendering pass (frontend)

**In the plan renderer** (`packages/web/src/plan/` — locate the canvas render loop):

When rendering plan elements, check the active view's `defaultLens`. Elements whose `discipline` does not match the lens render at `opacity: 0.25` using `var(--draft-witness)` as their stroke colour. `show_all` → all foreground. Elements with `discipline: null` are treated as `arch`.

**In the 3D viewport** (`packages/web/src/Viewport.tsx` or `meshBuilders.ts`):

Apply the same opacity pass: ghosted meshes render at 25 % opacity with `--draft-witness` material override. This is a CSS/WebGL uniform override, not a structural mesh change.

Implement `useLensFilter(viewId)` hook returning `(elem: BimElem) => 'foreground' | 'ghost'`. Used by both renderers.

## Implementation rules

- **No scope creep.** Do not implement LNS-V3-01 UI (status-bar dropdown) here — that is WP-050.
- **Ghost is opacity only, not hidden.** Elements in ghost state remain in the DOM / scene graph. Only opacity changes.
- **Design pillar D8.** Lens is a per-view modifier; changing it does not mutate any element's `discipline` field.
- **No hex literals** in `packages/web/src/**/*.tsx`. Use `var(--draft-witness)` for ghost colour.
