## Verbatim COL-V3-02 tracker entry (§6 T3, line 1421)

**COL-V3-02 — Permission tiers**

_Source: R-A §5. Severity High. Permission tiers without account-creation friction is the GTM unlock for B3._

**Status.** `now`.
**Scope.** Four roles: **admin**, **editor**, **viewer**, **public-link-viewer**. Enforced at three layers: (a) UI gates; (b) API gates; (c) yjs origin tagging. Public-link viewer: client clicks a link, comments, posts, never creates an account.

**Data model.**
```ts
type RoleAssignment = {
  modelId: string;
  subjectKind: 'user' | 'public-link';
  subjectId: string;
  role: 'admin' | 'editor' | 'viewer' | 'public-link-viewer';
  grantedBy: string;
  grantedAt: number;
  expiresAt?: number;
};
```

**Engine.** Server-side `authorize_command(userId, modelId, command)`. Public-link token in URL → synthetic `subjectId` + role. yjs orchestrator filters incoming ops by role.

**UI.** Share modal: three tabs — Workspace members / Invite by email / Public link. Participant strip shows roles via glyphs (admin=solid dot, editor=ring, viewer=hairline ring, public-link=dashed ring).

**Effort.** M — 1.5 weeks.

### 2. Pydantic schema (`app/bim_ai/permissions.py`)

New file:

```python
from __future__ import annotations
from typing import Literal
from pydantic import BaseModel, ConfigDict, Field

Role = Literal["admin", "editor", "viewer", "public-link-viewer"]
SubjectKind = Literal["user", "public-link"]

VERB_ALLOWLIST: dict[Role, set[str]] = {
    "admin":              {"*"},   # all verbs
    "editor":             {"createWall", "createDoor", "createWindow", "createStair",
                           "createLevel", "moveElement", "setWallStack", "assignElementToOption",
                           # add all create/mutate command types here
                           "createComment", "resolveComment", "createMarkup"},
    "viewer":             {"createComment", "resolveComment", "createMarkup"},
    "public-link-viewer": {"createComment", "resolveComment"},
}

class RoleAssignment(BaseModel):
    model_config = ConfigDict(populate_by_name=True, extra="ignore")
    id: str
    model_id: str = Field(alias="modelId")
    subject_kind: SubjectKind = Field(alias="subjectKind")
    subject_id: str = Field(alias="subjectId")
    role: Role
    granted_by: str = Field(alias="grantedBy")
    granted_at: int = Field(alias="grantedAt")
    expires_at: int | None = Field(default=None, alias="expiresAt")

def authorize_command(role: Role, command_type: str) -> bool:
    """Return True if the role is permitted to execute command_type."""
    allowed = VERB_ALLOWLIST.get(role, set())
    return "*" in allowed or command_type in allowed
```

### 3. API gate (`routes_api.py`)

In `apply_bundle_route`, resolve the caller's role and reject unauthorized commands:

```python
# Resolve caller role (stub for now — always 'admin' if no role record exists)
caller_role = await resolve_caller_role(session, model_id, body.user_id or "local-dev")
for cmd in body.bundle.commands:
    if not authorize_command(caller_role, cmd.type):
        raise HTTPException(
            status_code=403,
            detail=f"Role '{caller_role}' is not permitted to execute '{cmd.type}'"
        )
```

Add `resolve_caller_role` as an async helper that queries `role_assignments`; returns `"admin"` if no record found (backwards compat with existing local-dev flows).

### 4. REST share routes (`routes_api.py`)

Add three routes:

- `GET /models/{model_id}/roles` — list role assignments for the model.
- `POST /models/{model_id}/roles` — grant a role (body: `{subjectKind, subjectId, role}`). Admin only.
- `DELETE /models/{model_id}/roles/{assignment_id}` — revoke. Admin only.
- `POST /models/{model_id}/public-link` — create a public link token. Admin only. Returns `{token, url}`.

### 5. Public-link token resolution

When a request arrives with a `?token=<value>` query param (or `Authorization: Bearer <token>` where the value matches a public-link record), resolve to `subjectKind='public-link'` with role `'public-link-viewer'`. Store token in `RoleAssignmentRecord` with `subjectKind='public-link'` and `subjectId=<token>`.

## Implementation rules

- **Backwards compat.** If no `RoleAssignment` record exists for a caller, grant `admin` (dev default). This preserves all existing tests.
- **VERB_ALLOWLIST is the source of truth.** Do not hard-code role checks inline — always call `authorize_command`.
- **Public link token.** Use `secrets.token_urlsafe(32)` for token generation.
- **No hex literals.** CSS token variables only in TSX.
- **Additive schema.** New tables only — no modifications to existing tables.
