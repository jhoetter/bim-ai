## Verbatim CAN-V3-02 tracker entry

**Scope.** Canon hatches: brick at 45° offset, concrete dotted, insulation curve-stroked, plaster cross-hatched, timber grain striped, gypsum dotted-finer, stone irregular-poly. Hatches scale with paper-mm at the active plot scale — non-negotiable.

**Data model.** `{ kind: 'hatch_pattern_def'; id; name; paperMmRepeat; rotationDeg; strokeWidthMm; patternKind: 'lines' | 'crosshatch' | 'dots' | 'curve' | 'svg'; svgSource?; }`.

**Engine.** `screenRepeat = paperMmRepeat * (1/scaleDenom) * zoomFactor`. Every wall/floor/detail_region with a hatchId resolves through this.

**UI.** Material editor gains "Plan hatch" picker. Detail-region OptionsBar picks from the same library.

**Acceptance.** 1:50 plan with brick wall; hatch repeats at 73 mm × scale. Switch to 1:100; hatch repeats at half screen density but same brick rhythm in real-world mm. Hatches do NOT rescale on viewport zoom.

**Effort.** M — 1.5 weeks.

### 4. Hatch renderer (`packages/web/src/plan/HatchRenderer.ts`)

Core function:

```ts
export function computeHatchScreenRepeat(
  paperMmRepeat: number,
  plotScaleDenominator: number,  // e.g. 50 for 1:50
  viewportZoom: number,          // canvas zoom factor (1 = 100%)
): number {
  // Zoom affects screen pixels but NOT paper scale.
  // At 1:50, 1 paper-mm = 1000/50 = 20 real-mm on paper → scaled to screen pixels by zoom.
  const pixelsPerMm = viewportZoom * (96 / 25.4); // 96 dpi base
  return (paperMmRepeat / plotScaleDenominator) * pixelsPerMm;
}
```

Render functions per `patternKind` using SVG `<pattern>` elements:

- `lines`: parallel lines at `rotationDeg`, spacing = `screenRepeat`
- `crosshatch`: two sets of parallel lines at 45° each
- `dots`: dot grid, radius = `strokeWidthMm * pixelsPerMm / 2`
- `curve`: sinusoidal stroke (insulation symbol) along x-axis
- `svg`: embed `svgSource` as a raw SVG pattern tile

Use `<defs><pattern id="hatch-{id}" ...>` injected into the plan SVG layer. Apply via `fill="url(#hatch-{id})"` on closed region paths.
