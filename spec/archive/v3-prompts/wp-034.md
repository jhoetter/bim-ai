## Verbatim COL-V3-03 tracker entry (§6 T3, line 1445)

**COL-V3-03 — Shareable public link**

**Status.** `now`.
**Scope.** A model can be published as a public URL (revocable, optional expiry, optional password). URL renders the canvas in `public-link-viewer` mode. Always-current — refreshing shows the latest model state. Revocable instantly. Per-link analytics: which model, who opened it, when.

**Data model.**
```ts
type PublicLink = {
  id: string;
  modelId: string;
  token: string;                     // opaque URL fragment
  createdBy: string;
  createdAt: number;
  expiresAt?: number;
  passwordHash?: string;
  isRevoked: boolean;
  displayName?: string;
};
```

**Engine.** `POST /api/public-links/:modelId` creates; `GET /shared/:token` resolves to a synthetic session. Revocation flips `isRevoked`. Optional password is bcrypt-hashed. Analytics: log open events; surface in share modal as "Opened 3 times."

**Effort.** M — 1 week.

### 2. Pydantic schema (`app/bim_ai/public_links.py`)

New file:

```python
from __future__ import annotations
import secrets
from pydantic import BaseModel, ConfigDict, Field

class PublicLink(BaseModel):
    model_config = ConfigDict(populate_by_name=True, extra="ignore")
    id: str
    model_id: str = Field(alias="modelId")
    token: str
    created_by: str = Field(alias="createdBy")
    created_at: int = Field(alias="createdAt")
    expires_at: int | None = Field(default=None, alias="expiresAt")
    is_revoked: bool = Field(default=False, alias="isRevoked")
    display_name: str | None = Field(default=None, alias="displayName")
    open_count: int = Field(default=0, alias="openCount")

def generate_link_token() -> str:
    return secrets.token_urlsafe(32)

def verify_link_password(plain: str, hashed: str) -> bool:
    import bcrypt
    return bcrypt.checkpw(plain.encode(), hashed.encode())

def hash_link_password(plain: str) -> str:
    import bcrypt
    return bcrypt.hashpw(plain.encode(), bcrypt.gensalt()).decode()
```

If `bcrypt` is not in `pyproject.toml`, add it: `uv add bcrypt`.

### 3. REST routes (`app/bim_ai/routes_api.py`)

Add five routes:

- `POST /models/{model_id}/public-links` — create a link (admin only). Body: `{displayName?, expiresAt?, password?}`. Generate token via `generate_link_token()`. Also inserts a `RoleAssignment` record with `subjectKind='public-link'`, `subjectId=token`, `role='public-link-viewer'`.
- `GET /models/{model_id}/public-links` — list all non-revoked links with `openCount`.
- `POST /models/{model_id}/public-links/{link_id}/revoke` — set `isRevoked=True`; also deletes the corresponding `RoleAssignment`.
- `GET /shared/{token}` — resolve token → check not revoked + not expired → increment `openCount` → return the model's document (viewer-mode JSON; strip server-only fields).
- `POST /shared/{token}/verify-password` — body `{password}` → return `{ok: bool}`.

## Implementation rules

- **Token generation.** Always use `secrets.token_urlsafe(32)`. Never use `uuid4` for link tokens (predictable entropy).
- **Password is optional.** If no password is provided, `passwordHash` is null and the `verify-password` endpoint returns `{ok: true}` unconditionally.
- **Expiry is server-side.** The `GET /shared/{token}` resolver checks `expiresAt` server-side. Client-side checks are cosmetic only.
- **openCount increment is best-effort.** Use a simple `UPDATE ... SET open_count = open_count + 1`; do not make the resolve route depend on its success.
- **Role assignment on create.** Creating a public link must also create a `RoleAssignment` record. Revoking must delete it. This is what gives the public-link token access through COL-V3-02's `authorize_command` gate.
- **No hex literals.** CSS token variables only in TSX.
