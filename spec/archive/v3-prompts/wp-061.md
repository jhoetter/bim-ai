## Implementation tasks

### 1. Hide inspector when nothing selected

Find the Inspector / right-rail component (likely `packages/web/src/workspace/Inspector.tsx` or similar). Change the render condition so the rail is entirely absent from the DOM when no element is selected:

```tsx
const selectedIds = useStore(s => s.selectedIds);
if (selectedIds.length === 0) return null;  // not in DOM
```

Do not use `visibility: hidden` or `opacity: 0` — the element must not be in the DOM at all (`display: none` or conditional render).

### 2. Slide-in animation

When the selection changes from empty → non-empty, animate the inspector sliding in from the right:

```css
.inspector-rail {
  width: 300px;
  animation: inspector-slide-in 200ms var(--ease-paper) both;
}

@keyframes inspector-slide-in {
  from { transform: translateX(100%); opacity: 0; }
  to   { transform: translateX(0);    opacity: 1; }
}
```

The animation runs each time the inspector re-enters the DOM. On re-selection (one element → another), re-trigger by unmounting + remounting (key prop on selection id).

### 3. "Applies to: this / all N" radio

In the Inspector, when multiple elements of the same kind are in the model (e.g. 12 walls), add a small segmented radio at the top of the inspector:

```
○ This wall    ○ All 12 walls
```

- "This" (default): edits apply to the selected element only.
- "All N": clicking dispatches the edit command to all elements of that kind.
  - When "all N" is clicked, surface a non-blocking toast: "Next edit applies to all N walls."
  - After the edit is committed, revert to "This" automatically.

Count N from the element store. If there is only 1 element of that kind, omit the radio entirely.

### 4. Drawer for type / material edits (not modal)

When the user clicks a "Type" or "Material" field in the Inspector that would currently open a modal:

- Instead, open a **drawer overlay** that slides in from the right edge of the inspector rail.
- The drawer is 300 px wide, positioned over the inspector (not over the canvas).
- The canvas remains interactive (not blocked).
- Clicking outside the drawer closes it.
- The drawer renders the type/material picker list.

If a modal is currently used: replace it with this drawer pattern. If no modal exists yet: implement the drawer shell ready for future type/material pickers to use.

### 5. No hex literals

No `#xxxxxx` literals in new or modified TSX/CSS files. All colours via `var(--*)` tokens.

## Implementation rules

- **Frontend-only.** No Python changes, no schema changes, no index.ts changes.
- **Not in DOM when empty.** This is a hard requirement — `display: none` fallback is not acceptable; the element must be absent from the DOM.
- **No modal.** Type/material drawers must not block the canvas.
- **Additive.** Do not remove or restyle existing Inspector fields; only add the slide-in, the "all N" radio, and the drawer pattern.
