## Verbatim OUT-V3-01 tracker entry (§6 T4, line 1812)

**OUT-V3-01 — Live presentation URL — refreshable + revocable + public-link viewer**

**Status.** `next`. Ships after T3 link-share infra + CQ-01 lands. The B2 marquee output and Rayon copy item #8.

**Scope.** Hit "Share live presentation" → refreshable, revocable URL on clipboard; public viewer opens in a browser without an account; shows current Pages / Frames / Views state of the canvas; refreshes whenever the source canvas changes (websocket-driven). "Revoke" rotates the URL; existing readers see "presentation revoked." Per R-A §5: 10 000+ presentations published since soft launch — a top-of-funnel growth lever.

**Data model.** `{ kind: 'presentation_link'; id; modelId; pageScopeIds; token; permission: 'viewer'; allowMeasurement; allowComment; expiresAt?; createdAt; revokedAt? }`.

**Engine.** `POST /api/presentations/:modelId` issues a token-scoped read-only handle. Public route `GET /p/:token` resolves to a server-rendered viewer; viewer subscribes to a server-side websocket with the read-only token; updates stream as the canvas changes (CQ-01 makes this reliable). Rotating the token invalidates the WS session.

**UI.** Top bar → "Share live presentation" (inactive until at least one Page exists). Modal: scope chooser, measurement / comment toggles, optional expiry. "Copy link." Existing presentations list with "Revoke" per row. Public viewer is its own React route with no chrome editing affordances — only navigation, measurement, comment if enabled.

**Acceptance.** Create a 5-page Pages bundle; click "Share live presentation"; URL drops to clipboard; open in incognito → presentation visible without account; edit a wall on the source canvas → viewer refreshes within 500 ms; click "Revoke" → incognito viewer shows "presentation revoked" in <1 s. Brand-swap CI invariant on the public viewer (only Layer C changes).

**Effort.** L — 2–3 weeks. CQ-01 hard prerequisite.

**Cross-theme references.** T3 link-share infra (COL-V3-03 — hard prerequisite, now done); CQ-01 WebSocket robustness (hard prerequisite, now done); OUT-V3-02 (PPTX export walks same Frames state); OUT-V3-03 (branded PDF export shares same viewer chrome); D10 (Output is a feature, not an afterthought).

**Source.** R-A §5 + §6 (live presentation URL as top-of-funnel growth lever, 10k+ published); R-B §7 + §9 #5 (link-shareable canvas, frictionless packaging); COL-V3-03 infrastructure.

## Implementation tasks

### 1. Data model — Python (`app/bim_ai/document.py`)

Add `PresentationLink` to the `Document.elements` union:

```python
class PresentationLink(BaseModel):
    model_config = ConfigDict(populate_by_name=True, extra="ignore")
    kind: Literal["presentation_link"] = "presentation_link"
    id: str
    model_id: str = Field(alias="modelId")
    page_scope_ids: list[str] = Field(default_factory=list, alias="pageScopeIds")
    token: str
    permission: Literal["viewer"] = "viewer"
    allow_measurement: bool = Field(default=False, alias="allowMeasurement")
    allow_comment: bool = Field(default=False, alias="allowComment")
    expires_at: int | None = Field(default=None, alias="expiresAt")
    created_at: int = Field(alias="createdAt")
    revoked_at: int | None = Field(default=None, alias="revokedAt")
```

### 3. API routes (`app/bim_ai/routes_api.py`)

Add three new routes under `api_router`:

- `POST /models/{model_id}/presentations` — create a `PresentationLink`; generate token via `generate_link_token()` from `public_links.py`; persist as a `PublicLinkRecord` with `display_name="presentation"`; return `{ token, url }`.
- `GET /models/{model_id}/presentations` — list non-revoked presentation links for the model.
- `POST /models/{model_id}/presentations/{link_id}/revoke` — set `is_revoked=True`; return `{ revokedAt }`.

Public viewer route (under a separate router, no auth required):

- `GET /p/{token}` — look up `PublicLinkRecord` by token; if `is_revoked` return `{ status: 'revoked' }`; else return model snapshot + websocket upgrade URL. Increment `open_count`.

### 4. WebSocket session for viewer (`app/bim_ai/routes_api.py` or new `app/bim_ai/presentation_ws.py`)

- `WS /p/{token}/ws` — validate token; if revoked close with code 4403 + message `"presentation revoked"`; else subscribe the socket to the model's existing CQ-01 change-broadcast bus. When the source canvas changes, push a `{ type: 'snapshot_update' }` message. Token rotation (revoke) must push `{ type: 'revoked' }` and close all active WS sessions for that token.

### 5. "Share live presentation" button (`packages/web/src/workspace/TopBar.tsx`)

- Add a "Share live presentation" button to the right cluster of the top bar.
- Button is inactive (disabled + tooltip "Add a Page first") when no `Page` elements exist in the model.
- On click: open `SharePresentationModal`.
- Use `--color-surface-2` background, `--text-sm` label, no hex literals.

### 6. `SharePresentationModal` (`packages/web/src/collab/SharePresentationModal.tsx`)

Modal sections:
- **Scope chooser** — checkbox list of Pages; "All pages" is the default. Unchecked pages are excluded from `pageScopeIds`.
- **Toggles** — "Allow measurement" and "Allow comment" (off by default).
- **Optional expiry** — date picker or "No expiry."
- **Copy link** — calls `POST /models/:id/presentations`; puts URL on clipboard; shows "Copied!" toast.
- **Existing presentations** — table of active links (display name, created date, open count); "Revoke" button per row calls `POST /models/:id/presentations/:linkId/revoke`.

No hex literals. Use `--color-{surface,border,muted-foreground}` tokens throughout.

### 7. Public viewer route (`packages/web/src/viewer/PresentationViewer.tsx`)

A standalone React route (`/p/:token`):
- On mount: `GET /p/:token` → if `{ status: 'revoked' }` render "Presentation revoked" full-page message; else render canvas in read-only mode.
- Open `WS /p/:token/ws`; on `snapshot_update` message refresh canvas state; on `revoked` message show "Presentation revoked" overlay without page reload.
- No editing affordances (no top-bar buttons, no Inspector, no command palette).
- Viewer banner at top (adapt `PublicLinkBanner.tsx` pattern): "Shared by [author]" left side; "Sign in to edit →" right side.
- Navigation, measurement (if `allowMeasurement`), and comment (if `allowComment`) are the only enabled surfaces.
- Brand-swap CI invariant: only Layer C (`--brand-*`) tokens may differ from the default render.

### 8. Tests (`app/tests/test_out_v3_01.py`)

Cover:

- `POST /models/:id/presentations` creates a record; response contains `token` and `url`.
- `GET /p/:token` returns model snapshot (HTTP 200) when token is valid.
- `GET /p/:token` returns `{ status: 'revoked' }` (HTTP 200 with revoked payload) when `is_revoked=True`.
- `POST /models/:id/presentations/:linkId/revoke` sets `is_revoked=True`.
- `GET /models/:id/presentations` lists only non-revoked links.
- `pageScopeIds` empty list is accepted (treated as "all pages").
- `open_count` increments on each `GET /p/:token` resolve.
- WS connection closes with message `"presentation revoked"` after revoke is called.

## Implementation rules

These are verbatim from §A of the tracker:

- **Agent-callable contract per API-V3-01.** Every new REST endpoint must have a JSON schema. If you add CLI support, add it to `packages/cli/cli.mjs` + `app/bim_ai/commands.py`.
- **No scope creep.** OUT-V3-01 is the _link + viewer_ only. PPTX export is OUT-V3-02. Branded PDF templates are OUT-V3-03. Do not start those here.
- **Design-pillar adherence.** D10 (Output is a feature): the presentation URL is in the same code path as the canvas, not a separate exporter. D5 (Motion conveys causality): the refresh animation on `snapshot_update` uses `--motion-*` + `--ease-paper`, never a hard swap. D6 (Comment first-class): if `allowComment=true`, the comment tool is available in the viewer without any extra config.
- **No hex literals in `packages/web/src/**/*.tsx`.** The ESLint rule enforces this. Use `--color-*`, `--brand-*`, `--draft-*`, or `--text-*` tokens exclusively. The brand-swap CI test (`pnpm test brand-swap`) must stay green.
- **Token rotation invalidates WS.** When revoke is called, the server must push `{ type: 'revoked' }` on all open WS sessions for that token before closing them. The viewer must show the "Presentation revoked" state within 1 s.
