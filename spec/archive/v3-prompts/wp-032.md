## Verbatim VER-V3-02 tracker entry (§6 T3, line 1397)

**VER-V3-02 — Named milestones (Cmd+S snapshot dialog)**

**Status.** `now`.
**Scope.** Hit `Cmd+S` → snapshot dialog appears: "Name this milestone." Input commits to a `Milestone` entity that pins to the current snapshot id. Milestones surface as bold rows in VER-V3-01's activity stream and as a separate left-rail "Milestones" list. Restorable in two clicks. Named milestones survive forever (unlike auto-snapshots which can be GC'd). Maps onto git tag semantics without exposing git vocabulary.

**Data model.**
```ts
type Milestone = {
  id: string;
  modelId: string;
  name: string;                      // "Pre-client review v1"
  description?: string;
  snapshotId: string;
  authorId: string;
  createdAt: number;
};
```

**Engine.** `Cmd+S` intercepts the browser save shortcut → dialog. On submit, server creates a `Milestone` row + ensures the snapshot is GC-immune. Milestone rows in the activity stream render at `--text-xs` (one step heavier) with a flag glyph. Restore: clicking a milestone time-travels to its snapshot then offers "Restore to this version."

**Effort.** M — 1 week.

### 7. Wire `Cmd+S` keybinding (`packages/web/src/workspace/Workspace.tsx`)

Add a `useEffect` in `Workspace.tsx` that intercepts `Cmd+S` / `Ctrl+S` and opens the `MilestoneDialog`. Prevent the browser's default save-page behavior.

```ts
useEffect(() => {
  const handler = (e: KeyboardEvent) => {
    if ((e.metaKey || e.ctrlKey) && e.key === 's') {
      e.preventDefault();
      openMilestoneDialog();
    }
  };
  window.addEventListener('keydown', handler);
  return () => window.removeEventListener('keydown', handler);
}, [openMilestoneDialog]);
```

## Implementation rules

- **`snapshotId` is the model's current revision** (an integer cast to string is fine for v3). The real snapshot-id system is a v3.1 enhancement.
- **Activity row on create.** Every `POST /milestones` must call `emit_activity_row` with `kind='milestone_created'`. This makes milestones appear in the activity stream automatically.
- **`Cmd+S` prevention.** `e.preventDefault()` must be called to prevent the browser from opening a save-file dialog on unsaved documents.
- **Additive schema.** `milestones` table is new; no changes to existing tables.
- **No hex literals.** CSS token variables only in TSX.
