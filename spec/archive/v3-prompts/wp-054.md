## Verbatim MAT-V3-01 tracker entry

#### MAT-V3-01 — Material PBR map slots + decals

**Scope.** Extend Material to optional PBR map slots: `albedoMapId`, `normalMapId`, `roughnessMapId`, `metallicMapId`, `heightMapId`. Each map is an ImageAsset referenced by id. Add `Decal` element kind: 2D image hosted to a parent surface with a rectangle of UV space. Materials render in 3D viewport with PBR shader; in plan/elevation, materials render as albedo or hatch. Decals render only in 3D and rendered output.

**Data model.**
```ts
{
  kind: 'material';
  albedoColor?: string;
  albedoMapId?: string;
  normalMapId?: string;
  roughnessMapId?: string;
  metallicMapId?: string;
  heightMapId?: string;
  uvScaleMm?: { uMm: number; vMm: number };
  uvRotationDeg?: number;
  hatchPatternId?: string;
}
{
  kind: 'decal';
  parentElementId: string;
  parentSurface: 'front' | 'back' | 'top';
  imageAssetId: string;
  uvRect: { u0, v0, u1, v1 };
}
```

**Effort.** M — 1 week.

### 5. 3D viewport PBR rendering (`packages/web/src/viewport/meshBuilders.ts`)

When building a mesh for a wall/floor/element that has a material with PBR map slots:

- If `albedoMapId` is set, fetch the image asset by id from the model's `imageAssets` store and apply as the albedo texture
- If `normalMapId` is set, apply as normal map
- For `roughnessMapId` and `metallicMapId`, apply to the PBR material's roughness/metallic channels
- UV scaling: apply `uvScaleMm` to control the texture repeat. Default: `{ uMm: 1000, vMm: 1000 }` (1 m repeat).
- For v3, a simplified PBR shader is sufficient: albedo texture + normal map + roughness scalar. Full PBR will be refined in T5.

For `DecalElem`: render as a projected plane in 3D at the specified `uvRect` on the parent surface. Use a `MeshBasicMaterial` with the image asset texture and `transparent: true`.
