## Background
CHR-V3-07 refreshes the left-rail project browser tree. Currently a flat list or minimal tree; v3 ships:
- 240 px expanded / 36 px collapsed widths
- Structured groups: Views, Schedules, Links/Imports, Discipline groups (auto-derived), Phases
- Drag-to-reorder views
- Right-click context menu: duplicate / template-create / rename / delete
- Search at top

Zero hex literals throughout — CSS vars only.

## Implementation tasks

### 1. Locate existing project browser

Search for `ProjectBrowser` or similar in `packages/web/src/workspace/`. If it exists, extend it. If not, create `packages/web/src/workspace/ProjectBrowser.tsx`.

### 2. Component structure (`ProjectBrowser.tsx`)

```tsx
// ProjectBrowser — left-rail tree at 240px expanded / 36px collapsed
// Width: var(--rail-width-expanded, 240px) / var(--rail-width-collapsed, 36px)
// Zero hex literals

type ProjectBrowserProps = {
  elements: Element[];
  activeViewId: string | null;
  onActivateView: (viewId: string) => void;
  onRenameView: (viewId: string, newName: string) => void;
  onDeleteView: (viewId: string) => void;
  onDuplicateView: (viewId: string) => void;
  collapsed?: boolean;
};
```

Groups to render (derive from elements):
- **Views** — all `viewpoint` + `saved_view` elements, grouped by discipline tag if present
- **Schedules** — all `schedule_view` elements  
- **Links / Imports** — `image_underlay` + DXF underlay elements
- **Phases** — phase elements if any

Each group header: `--text-sm` in `text-eyebrow-tracking` CSS class (uppercase small tracking). Rows: `--text-sm`.

### 3. Right-click context menu

When a view row is right-clicked, show a small context menu with:
- Rename
- Duplicate
- Delete

Use an existing context menu pattern from the codebase, or create a minimal `ContextMenu` component.

### 4. Drag-to-reorder

Views within a group can be dragged to reorder. On drop, dispatch a `ReorderView` command (or store action). Use the HTML5 drag-and-drop API (no extra deps).

### 5. Search

A `<input type="search">` at the top of the panel. Filters rows in real-time (case-insensitive name match). When search is active, groups collapse to show only matching items.

### 6. New `ReorderViewCmd` in core types

Add to `packages/core/src/index.ts`:
```typescript
export type ReorderViewCmd = {
  type: 'reorder_view';
  viewId: string;
  newSortOrder: number;
};
```

Add to `Command` union.

Add corresponding Python command in `app/bim_ai/commands.py` + engine handler (no-op handler is fine — just set a `sort_order` field on the element).

### 7. CSS tokens (`packages/web/src/tokens-drafting.css`)

If not already present, add:


### 8. Tests (`packages/web/src/workspace/ProjectBrowser.test.tsx`)

Write at least 10 tests:

- Drag-to-reorder emits correct new order
