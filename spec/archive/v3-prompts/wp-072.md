## Background
OUT-V3-03 ships a `BrandTemplate` element and the brand-aware export contract. PDF + PPTX exporters accept a `brandTemplateId`. Locked decision: brand overrides are CSS/Layer-C only — no component changes; no kernel geometry changes. Export writes the file with brand applied (chrome accents, typeface, logo) without touching Layer A (drafting palette, line weights, motion).

### 2. Engine handlers (`app/bim_ai/engine.py`)

Standard CRUD for `BrandTemplateElem`.

Validation:
- `accent_hex` must match `^#[0-9a-fA-F]{6}$`
- `accent_foreground_hex` must match same pattern
- `css_override_snippet` is accepted as-is (no validation — CSS is opaque)

### 3. Brand-aware export endpoint (`app/bim_ai/routes_api.py`)

Add `GET /api/v3/models/{model_id}/export/pdf?brandTemplateId=<id>` — returns a `BrandedExportBundle` JSON:
```python
# BrandedExportBundle
{
    "schemaVersion": "out-v3.0",
    "format": "pdf",
    "brandTemplateId": str | None,
    "brandLayer": {
        "accentHex": str,
        "accentForegroundHex": str,
        "typeface": str,
        "logoMarkSvgUri": str | None,
        "cssOverrideSnippet": str | None,
    } | None,
    "sheets": [...],  # list of {sheetId, name}
    "invariantCheck": "layer-c-only"  # brand-swap invariant signal
}
```

If `brandTemplateId` is provided but not found → 404. If not provided → `brandLayer: null`.

### 4. Brand-swap invariant test helper (`app/tests/api/test_out_v3_03.py`)

The brand-swap invariant: only Layer C tokens differ on brand swap, never Layer A. Write a utility assertion `assert_brand_only_layer_c(bundle: dict)` that checks `bundle["invariantCheck"] == "layer-c-only"`.
