## Verbatim LNS-V3-01 tracker entry

#### LNS-V3-01 — Lens dropdown in status bar

**Status.** `now`. Antidote to A8.
**Scope.** Status-bar dropdown "Show: ▾" — 4-item menu: Architecture / Structure / MEP / All. Click flips `defaultLens` on the active view only. Foreground/ghost transition runs `--ease-paper` over 240 ms. Keyboard `L` cycles forward.
**Data model.** No new fields; reuses `defaultLens` from DSC-V3-02.
**Engine.** `SetViewLens(activeViewId, lensMode)` — undoable, activity-stream-aware.
**UI.** Right side of status bar, left of zoom indicator. Chip reads "Show: Architecture ▾." Check-mark on active. When lens ≠ `show_all`, chip's left edge tints `--disc-*-soft`.
**Acceptance.** Click "Show: ▾" → "Structure." In 240 ms arch walls fade to 25% ghost. Click "All" → everything returns. Two clicks vs Revit's 120-toggle dialog.
**Effort.** S — 4 days.

export type LensMode = 'show_arch' | 'show_struct' | 'show_mep' | 'show_all';

These will be deduped when the orchestrator merges DSC-V3-02 first.

### 2. `LensDropdown` component (`packages/web/src/workspace/LensDropdown.tsx`)

```tsx
const LENS_LABELS: Record<LensMode, string> = {
  show_arch: 'Architecture',
  show_struct: 'Structure',
  show_mep: 'MEP',
  show_all: 'All',
};

const LENS_CYCLE: LensMode[] = ['show_arch', 'show_struct', 'show_mep', 'show_all'];

export function LensDropdown({ viewId, currentLens, onChangeLens }: {
  viewId: string;
  currentLens: LensMode;
  onChangeLens: (lens: LensMode) => void;
}) { ... }
```

- Renders a chip: `Show: {LENS_LABELS[currentLens]} ▾`
- When `currentLens !== 'show_all'`, left border tints with `var(--disc-arch-soft)` / `var(--disc-struct-soft)` / `var(--disc-mep-soft)` as appropriate
- Click opens a 4-row menu (using existing dropdown primitive — no custom portal unless one exists)
- Each row shows the lens label + a check-mark SVG glyph when active
- Select row → calls `onChangeLens(lens)` + closes menu
- Menu closes on outside click or `Esc`
- `transition: opacity 240ms var(--ease-paper)` on the ghost elements is handled by the renderer (DSC-V3-02); this component only dispatches the command
- **No hex literals.** Every colour via `var(--*)` token.

### 3. Keyboard shortcut `L` (`packages/web/src/workspace/LensDropdown.tsx` or global keymap)

Wire `L` key (when no input is focused and no draw tool is active) to cycle through `LENS_CYCLE` on the active view:

```ts
const handleKeyDown = (e: KeyboardEvent) => {
  if (e.key === 'l' || e.key === 'L') {
    if (document.activeElement?.tagName === 'INPUT') return;
    const idx = LENS_CYCLE.indexOf(currentLens);
    onChangeLens(LENS_CYCLE[(idx + 1) % LENS_CYCLE.length]);
  }
};
```

Register via `useEffect` on `window`. Deregister on unmount.

### 4. Wire into `StatusBar.tsx`

In `packages/web/src/workspace/StatusBar.tsx`, import `LensDropdown` and render it in the right slot (left of zoom indicator):

```tsx
<LensDropdown
  viewId={activeViewId}
  currentLens={activeView?.defaultLens ?? 'show_all'}
  onChangeLens={(lens) => dispatch({ type: 'set_view_lens', viewId: activeViewId, lens })}
/>
```

`dispatch` calls the existing command bus / store action → `POST /api/v3/models/{id}/apply` with `SetViewLensCmd`.

### 5. Activity stream entry

`SetViewLensCmd` dispatches an activity entry `{ kind: 'lens_changed', viewId, lens }` via the existing activity bus. No new activity type needed if a generic `view_updated` entry already exists; otherwise add a minimal `LensChangedEntry` to `activity.py`.
