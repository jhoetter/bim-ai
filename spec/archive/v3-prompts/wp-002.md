## WP-002 verbatim from tracker (§6 T2 EDT-V3-01)

**EDT-V3-01 — Constraint rules: parallel / perpendicular / collinear / equal_length**

**Status.** `now`.

**Scope.** Four geometric constraint evaluators that the engine calls after every command apply. Each evaluator receives the two `refsA` / `refsB` element groups and the current element dictionary; it returns a `residual_mm` (or `residual_deg` for angular rules) that the engine compares against `EPSILON_MM`. If any `error`-severity constraint fires, the command is rejected and the world rolls back.

- **parallel** — the two wall/grid-line direction vectors are parallel (cross-product magnitude ≤ EPSILON_MM per unit length). `residual` = cross-product magnitude.
- **perpendicular** — dot product of direction vectors ≤ EPSILON_MM. `residual` = |dot product|.
- **collinear** — two walls lie on the same infinite line (direction parallel AND offset between start points projected onto the normal ≤ EPSILON_MM). `residual` = max(angle residual, offset residual).
- **equal_length** — the two wall/beam lengths are equal (|len_a − len_b| ≤ EPSILON_MM). `residual` = |len_a − len_b|.

**Data model.** `ConstraintRule` literal already has `parallel | perpendicular | collinear | equal_length` entries in `commands.py`. The `ConstraintRefRow` already carries `elementId` + `edge: Literal["start", "end", "axis"] | None`. No schema changes needed.

**Engine.** Replace the four stub evaluators in `app/bim_ai/edt/constraints.py` with real implementations. Each must:
1. Extract direction vectors from `WallElem` or `GridLineElem` via `(end - start) / |end - start|`.
2. Compute the geometric residual.
3. Return a `ConstraintResult(residual_mm=..., ok=residual <= EPSILON_MM)`.

**UI.** `GripLayer.tsx` already renders a padlock icon for `equal_distance` constraints. Add padlock overlays for the four new rules using the same `--constraint-lock-colour` CSS token. No new icon needed — same padlock glyph, different rule label in the tooltip.

**Acceptance.**
- `pytest app/tests/test_constraint_rules.py` — all tests pass (write them: ≥ 4 × 3 = 12 tests covering ok + violation + edge cases for each rule).
- Engine rejects a `moveWallDelta` that would violate a `perpendicular` constraint with severity `error`.
- Engine allows the same move when constraint severity is `warning`.
- `parallel` and `collinear` evaluators handle zero-length input without crashing.

**Effort.** L — 2.5 weeks.

**Cross-theme references.** T2 (grip provider uses constraint state to show lock icon); T9 (CMD-V3-01 bundles that move geometry trigger constraint re-evaluation post-apply).

## Implementation tasks

1. **Read `constraints.py` end-to-end** — understand `ConstraintResult`, `evaluate_constraints`, and the existing `equal_distance` implementation as the reference pattern.

2. **Implement `parallel`** — extract direction vectors from both ref elements; compute cross-product magnitude; return `ConstraintResult(residual_mm=cross_mag, ok=cross_mag <= EPSILON_MM)`.

3. **Implement `perpendicular`** — compute |dot product| of normalised direction vectors; residual = |dot|.

4. **Implement `collinear`** — parallel check PLUS: project the vector between the two start points onto the shared normal; residual = max(angle_residual, offset_mm).

5. **Implement `equal_length`** — compute lengths via `hypot`; residual = |len_a - len_b|.

6. **Write `app/tests/test_constraint_rules.py`** — ≥ 12 tests: for each rule, test (a) ok case, (b) violation rejected, (c) edge case (zero-length, parallel walls with offset). Follow the style of existing engine tests.

7. **GripLayer.tsx** — add padlock overlay rendering for the four new rules. Look at how `equal_distance` padlocks are rendered and replicate for the new rules. Use the same CSS token.

## Implementation rules

- Do NOT change `ConstraintRule` in `commands.py` — the four literals are already there.
- Do NOT modify `ConstraintElem` in `elements.py` — schema is frozen.
- The evaluators must be **pure functions** — no side effects, no I/O.
- Zero-length walls must not crash: guard with `if length < EPSILON_MM: return ConstraintResult(residual_mm=0.0, ok=True)` (a zero-length wall satisfies any orientation constraint vacuously).
