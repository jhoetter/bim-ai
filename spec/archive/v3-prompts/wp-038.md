## Verbatim VIE-V3-03 tracker entry (¬ß6 T4, line 1702)

**VIE-V3-03 ‚Äî View templates with apply + edit + propagation toast (R-D G22 + R-D U6)**

**Status.** `now`.
**Scope.** Saved bundle of view filter + visibility-graphic overrides + scale + detail level + crop region + display style. Applied to N views. Editing the template propagates to all bound views with a non-modal toast ‚Äî _"42 views updated, 3 unbound."_ The difference between a 1-hour and a 5-hour drawing pass.

**Data model.** `{ kind: 'view_template'; id; name; scale?; detailLevel?; cropDefault?; visibilityFilters?; elementOverrides?; phase?; phaseFilter? }`. View gains optional `templateId`.

**Engine.** `CreateViewTemplate` / `UpdateViewTemplate` / `ApplyViewTemplate` / `UnbindViewTemplate`. On `UpdateViewTemplate`, kernel walks `views.filter(v => v.templateId === tpl.id)`, re-resolves their effective bundle, fires a typed `ViewTemplatePropagation { affected, unbound }` event for the toast.

**UI.** Project Browser ‚Üí "View Templates" group. Per-template Apply / Edit / Duplicate / Delete. Edit opens a panel mirroring a normal view's properties; on save fires the propagation event. Toast reads _"42 views updated, 3 unbound. View list."_ "View list" scopes the project browser to the affected views.

**Effort.** L ‚Äî 2 weeks.

### 3. Engine handlers (`app/bim_ai/engine.py`)

**`CreateViewTemplateCmd`** ‚Äî create a `ViewTemplate` element with the given id and fields.

**`UpdateViewTemplateCmd`** ‚Äî find the template by `templateId`; apply partial patch (only non-None fields). After patching: walk all `View` elements where `view.templateId == templateId`; for each bound view, apply the template fields to the view (scale, detailLevel, elementOverrides, phase overwrite from template where template value is not None). Return a `ViewTemplatePropagation` in the result payload:
```python
{
    "event": "ViewTemplatePropagation",
    "templateId": template_id,
    "affected": [v.id for v in bound_views],
    "unbound": []
}
```

**`ApplyViewTemplateCmd`** ‚Äî find the view; set `view.templateId = templateId`; propagate the template's non-None fields to the view immediately.

**`UnbindViewTemplateCmd`** ‚Äî set `view.templateId = None`. The view retains its current property values (they are no longer inherited from the template). Return a `ViewTemplatePropagation` with `affected=[]` and `unbound=[viewId]`.

**`DeleteViewTemplateCmd`** ‚Äî remove the `ViewTemplate` from `doc.elements`. All views bound to it get `templateId = None` (implicit unbind). Do not change their current property values.

## Implementation rules

- **Propagation is synchronous in the kernel.** `UpdateViewTemplate` walks all bound views in the same command handler and updates them immediately. There is no deferred propagation.
- **Template fields are sparse.** Only non-None template fields overwrite view fields. A template with `scale=None` does not touch the view's scale.
- **Unbind does not revert.** `UnbindViewTemplate` clears `templateId` only ‚Äî the view keeps its current scale / detailLevel values (which may have been set by the template).
- **Toast is non-modal.** The user can keep working; `PropagationToast` never blocks interaction.
- **No hex literals.** CSS token variables only in TSX.
- **Additive schema.** `ViewTemplate` is a new element kind; `View.templateId` is optional. Existing documents round-trip unchanged.
