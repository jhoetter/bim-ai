## Verbatim SCH-V3-01 tracker entry

#### SCH-V3-01 — Custom-properties + filterable spreadsheet view

**Scope.** Two coupled primitives. **(a)** Custom-properties: `PropertyDefinition` per project; element-level `props: Record<string, unknown>`. **(b)** Schedule view: filterable spreadsheet over a category; columns = built-in fields + custom-properties; rows = element instances. Filter/sort/group. Edit any cell → mutates element via command bus. Cross-highlight: hover row → element pulses in plan/3D.

**Data model.**
```ts
type PropertyDefinition = {
  key: string; label: string;
  kind: 'mm' | 'm2' | 'currency' | 'enum' | 'string' | 'bool' | 'date';
  enumValues?: string[];
  defaultValue?: unknown;
  appliesTo: Array<ElemKind | '*'>;
  showInSchedule: boolean;
};
type ScheduleView = {
  kind: 'schedule'; id: string; name: string;
  category: ElemKind;
  columns: Array<{ fieldKey: string; label: string; width?: number }>;
  filterExpr?: string;
  sortKey?: string; sortDir?: 'asc' | 'desc';
};
```

**Effort.** M — 1.5 weeks.

    type: Literal["create_schedule_view"] = "create_schedule_view"

    name: str
    category: str
    columns: list[dict] = Field(default_factory=list)

Add all three to the `Command` union. Engine handlers:

- `CreatePropertyDefinitionCmd` → stores `PropertyDefinitionElem`
- `SetElementPropCmd` → finds element by `element_id`, updates `props[key] = value` via `model_copy`
- `CreateScheduleViewCmd` → stores `ScheduleViewElem`

### 5. Schedule query endpoint (`app/bim_ai/routes_api.py`)

```
GET /api/v3/models/{modelId}/schedules/{scheduleId}/rows
```

Returns a JSON array of rows, one per element matching `scheduleView.category`. Each row:
```json
{
  "elementId": "...",
  "fields": {
    "id": "...",
    "name": "...",
    "lengthMm": 4200,
    "cost_per_m2": 85.0
  }
}
```

Built-in fields are derived from the element (e.g. `lengthMm` for walls, `widthMm` for doors). Custom-property fields come from `element.props`. Apply `filterExpr` (simple string match on any field value), `sortKey` / `sortDir`.

Requires `viewer` permission.

### 7. `ScheduleView` component (`packages/web/src/workspace/ScheduleView.tsx`)

Spreadsheet-style table component:

- Fetches rows from `GET /schedules/{id}/rows` on mount + on model update
- Renders `<table>` with sticky header row showing column labels
- Each data row: cell per column, renders value in `--text-sm` tabular-nums
- Click any editable cell → inline `<input>` → Enter dispatches `SetElementPropCmd` or the relevant update command for built-in fields
- Hover a row → dispatches a `HIGHLIGHT_ELEMENT` UI action (element pulses in plan canvas for 1.5 s)
- Filter bar at top: text input → re-fetches with `filterExpr`
- Sort: click column header → toggles asc/desc
- No hex literals. All colours via `var(--*)` tokens.

### 8. Activate from Project Browser / View menu

The schedule view appears in the project browser under "Schedules." Double-click → opens in the main canvas area as a tab. Alternatively, wire to a "New Schedule" menu item in the View menu.
