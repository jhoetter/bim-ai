## Verbatim EDT-V3-11 tracker entry (§6 T2, line 1131)

**EDT-V3-11 — Phase-filter as view-as-lens (R-D U7)**

**Status.** `now`. Anti-pattern A7 antidote (Revit's dual-dropdown phasing UX).

**Scope.** Each view (plan / elevation / section / 3D) gains a **phase dropdown** in its top chrome — "Existing," "Demolition," "New Construction," "Show All." Switching re-renders _without touching the model_. Existing: `--phase-existing` grey; demolition: grey-dashed; new: bold-black. The same dropdown pattern is reused verbatim for Design Options A/B/Main switching (T1 KRN-V3-04).

**Data model.** Each view gains `phaseFilter: 'existing' | 'demolition' | 'new' | 'all'` and `designOptionId: string | 'main'`. Defaults `all` / `main`. Persisted with the view doc.

**Engine.** Render-time predicate at scene-graph build: walks elements, applies both filters. Phase-render style is token-driven shader (line dash + colour). Kernel never sees the filter; only the renderer does.

**Acceptance.** Renovation seed model: Existing → grey solid walls only; Demolition → grey-dashed only; Show All → phase-coloured render. Same dropdown swapped to Design Option toggles Kitchen A ↔ B.

**Effort.** M — 1.5 weeks.

### 3. Engine — phase filter predicate (`engine.py`)

Add a pure function:

```python
def element_passes_phase_filter(
    elem: Any,
    phase_filter: str,
) -> bool:
    """Return True if the element should be visible under the given phase filter.

    Rules (using element.phase_created / element.phase_demolished):
    - 'all':        always visible.
    - 'existing':   phase_created == 'existing' and (phase_demolished is None or phase_demolished == 'existing')
    - 'demolition': phase_demolished == 'demolition'
    - 'new':        phase_created == 'new'
    """
    if phase_filter == "all":
        return True
    phase_created = getattr(elem, "phase_created", "existing")
    phase_demolished = getattr(elem, "phase_demolished", None)
    if phase_filter == "existing":
        return phase_created == "existing" and (phase_demolished is None)
    if phase_filter == "demolition":
        return phase_demolished == "demolition"
    if phase_filter == "new":
        return phase_created == "new"
    return True
```

### 4. Engine — phase render style (`engine.py`)

Add a pure function that returns the render-style tokens for a given phase/filter combination:

```python
def phase_render_style(
    elem: Any,
    phase_filter: str,
) -> dict[str, str]:
    """Return CSS token names for the element's phase render style.

    Returns a dict with keys 'stroke', 'strokeDashArray', 'strokeWidth'.
    """
    phase_created = getattr(elem, "phase_created", "existing")
    phase_demolished = getattr(elem, "phase_demolished", None)

    if phase_demolished == "demolition":
        return {
            "stroke": "var(--phase-demolition)",
            "strokeDashArray": "4 4",
            "strokeWidth": "var(--draft-lw-projection)",
        }
    if phase_created == "new":
        return {
            "stroke": "var(--phase-new)",
            "strokeDashArray": "none",
            "strokeWidth": "var(--draft-lw-cut)",
        }
    # existing
    return {
        "stroke": "var(--phase-existing)",
        "strokeDashArray": "none",
        "strokeWidth": "var(--draft-lw-projection)",
    }
```

Ensure `--phase-existing`, `--phase-demolition`, `--phase-new` are defined in the CSS token file (or add them if missing):
- `--phase-existing`: grey (e.g. `hsl(0 0% 60%)`)
- `--phase-demolition`: grey with dashed line
- `--phase-new`: bold black (`hsl(0 0% 10%)`)

## Implementation rules

- **Kernel never sees the filter.** `element_passes_phase_filter` and `phase_render_style` are renderer-layer functions. They do NOT mutate the document or affect any kernel element. They are pure read-time predicates.
- **Renderer, not engine.** Phase filter is applied at render time (when building the scene graph / plan mesh), not at command application time. Do not add phase-filter logic to `apply_inplace`.
- **Same dropdown pattern for design options.** The `PhaseDropdown` component should be parameterised enough that `KRN-V3-04`'s design-option dropdown (WP-024) can reuse the same shell.
- **Additive schema.** `phaseFilter` defaults to `'all'`; existing views without the field round-trip unchanged.
- **No hex literals.** All phase colours use CSS token variables (`--phase-*`).
- **Cross-test with detail level.** When `phase_filter='demolition'` and `detail_level='coarse'`, the rendering combines both: coarse geometry + dashed-grey style. Write one combined test for this.
