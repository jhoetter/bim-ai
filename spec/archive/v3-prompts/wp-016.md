## Verbatim KRN-V3-07 tracker entry (§6 T1, line 694)

**KRN-V3-07 — Slanted & tapered walls**

_Source: R-D §3 G3. Severity Medium → High residential. Lofts, attics, retaining walls._

**Status.** `next`.
**Scope.** Walls that lean (top-edge offset relative to base) or taper (top thickness ≠ base thickness). The kernel handles non-prismatic solids; openings follow lean (a window in a leaning wall stays in the wall plane).
**Data model.**
```ts
{
  kind: 'wall';
  // existing fields preserved
  leanMm?: { xMm: number; yMm: number };     // top-vs-base XY offset
  taperRatio?: number;                        // top thickness / base thickness; 1 = prismatic
}
```
**Engine.** Wall solid generation walks a non-prismatic profile (base rectangle + lean offset + taper). Hosted elements (doors, windows, openings) compute on the lean plane, not the vertical plane. Plan cut-plane intersects the slanted volume correctly; elevation paints the trapezoid honestly. Validation: lean ≤ wall height tan(60°); taper ratio ∈ (0.1, 10).
**UI.** Inspector adds "Lean" (Vec2 input) + "Taper ratio" (number); 3D viewport gains a top-edge handle that drags in XY for lean.
**Acceptance.** Author a 3 m wall with 200 mm lean to the south; place a window at mid-height; in 3D, the wall reads slanted; in elevation, the window stays flush with the wall plane; in plan, the cut at 1.2 m shows the wall offset by `lean × (1.2/3.0)`.
**Effort.** M — 1.5 weeks.
**Cross-theme references.** T2 (3D top-edge lean handle reuses EDT-03 grip protocol); T7 (retaining walls on toposolid).

### 1. Data model — Python (`elements.py`)

```python

```

Where `Vec2Mm` is the existing 2D point type (already defined in `elements.py`). Both fields default to `None` so existing walls round-trip unchanged.

Add a `@model_validator(mode="after")` on `WallElem` (or extend the existing `_validate_shape_specific_fields` if one exists) to enforce:

### 4. Engine validation helper (`engine.py`)

Add a private pure function near the `_validate_wall_stack` helper:

```python
import math

def _validate_wall_lean_taper(
    lean_mm: "Vec2Mm | None",
    taper_ratio: float | None,
    wall_height_mm: float,
) -> None:
    """Validate lean and taper parameters for a wall.

    Rules:
    - leanMm magnitude must be ≤ wall_height_mm * tan(60°).
    - taperRatio, if set, must be in the open interval (0.1, 10).
    """
    if lean_mm is not None:
        magnitude = math.sqrt(lean_mm.x_mm ** 2 + lean_mm.y_mm ** 2)
        max_lean = wall_height_mm * math.tan(math.radians(60))
        if magnitude > max_lean:
            raise ValueError(
                f"leanMm magnitude ({magnitude:.1f} mm) exceeds "
                f"wall height × tan(60°) ({max_lean:.1f} mm)"
            )
    if taper_ratio is not None:
        if not (0.1 < taper_ratio < 10.0):
            raise ValueError(
                f"taperRatio ({taper_ratio}) must be in open interval (0.1, 10)"
            )
```

### 7. Plan cut-plane offset helper (`engine.py` or new module)

Add a pure function to compute the wall's face offset at a given cut height, used by plan renderers:

```python
def resolve_wall_face_offset_at_cut(
    wall: "WallElem",
    cut_plane_offset_mm: float,
) -> tuple[float, float]:
    """Return the (x_offset_mm, y_offset_mm) of the wall's top face
    relative to its base face at the given cut height.

    For a prismatic wall, returns (0.0, 0.0).
    For a leaning wall, returns lean_mm * (cut_plane_offset_mm / wall.height_mm).
    """
    if wall.lean_mm is None or wall.height_mm == 0:
        return (0.0, 0.0)
    fraction = cut_plane_offset_mm / wall.height_mm
    return (
        wall.lean_mm.x_mm * fraction,
        wall.lean_mm.y_mm * fraction,
    )
```

Export this function so plan renderers and tests can import it.

## Implementation rules

- **API-V3-01 contract.** `SetWallLeanTaperCmd` is a new kernel command. It must be added to:
  1. `commands.py` (`Command` union).
  2. `engine.py` `apply_inplace` dispatch.
  3. Any tool-registry / descriptor file if one exists (check `app/bim_ai/api/registry.py`); add a comment noting the new command type is now valid if no explicit registration is required for the command-level.
  No separate CLI verb is required for this WP (the command is issued as part of a bundle).
- **No hex literals in chrome.** If any new React / TSX is added for the Inspector "Lean" and "Taper ratio" fields, all colours must use CSS design-token variables. No bare `#xxxxxx` strings in TSX or CSS files.
- **Schema is additive.** `WallElem.lean_mm` and `WallElem.taper_ratio` are optional with default `None`. No migration needed; existing walls without lean/taper round-trip unchanged.
- **Pydantic alias discipline.** All new Python fields use `alias=` (camelCase) and `populate_by_name=True` consistent with the rest of `elements.py`.
- **No changes to `ElementKind` or `ElemKind`** — `'wall'` already covers slanted/tapered walls.
- **Pure functions only.** `_validate_wall_lean_taper` and `resolve_wall_face_offset_at_cut` must be pure — no side effects, no I/O, no mutation of arguments.
- **Do not split `engine.py`.** CQ-04 is a separate WP. All new engine logic stays in `engine.py`; helpers may be extracted to a `wall_geometry.py` module if the file is already very large.
