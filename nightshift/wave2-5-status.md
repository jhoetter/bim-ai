### PLN-02 — Crop region UI toggle (`done`)

- `plan_view.cropEnabled` / `plan_view.cropRegionVisible` fields wired through
  `packages/core/src/index.ts`, `app/bim_ai/elements.py`, and the
  `updateElementProperty` plan-view setter in `app/bim_ai/engine.py`
  (parses `'true'/'false'/''`-as-inherit just like `planShowOpeningTags`).
- New pure helper `packages/web/src/plan/cropRegionDragHandles.ts`:
  `cropHandlePositions`, `applyCropHandleDrag`, `pickCropHandle`,
  `pointInsideCrop`, `cropDragCommands`. Math is independent of
  three.js so it stays unit-testable; min-span clamp prevents the frame
  from collapsing on aggressive drags.
- `PlanCanvas.tsx` renders a dashed crop frame + 8 corner/edge drag
  handles whenever the active plan view has crop bounds and either
  flag is set; pointer-down picks a handle (or the body for shift-drag),
  pointer-move replays the bounds via the helper into a refreshed
  overlay, pointer-up commits two `updateElementProperty` commands.
- `cropEnabled` clips per-mesh visibility post-`rebuildPlanMeshes` so
  toggling the checkbox makes outside walls / doors / windows / rooms
  vanish without touching the wire pipeline.
- View-properties panel ("Crop region") sits bottom-right with the two
  checkboxes; only renders when crop bounds exist on the active view.

### PLN-01 — Auto-dimension / auto-tag automation (`done`)

- New element kind `placed_tag` (host = room/door/window) with
  `auto_generated` flag; new commands `placeTag`,
  `clearAutoGeneratedAnnotations`, plus `autoGenerated:true` on
  `createDimension`. Engine validates host kinds and view kinds.
- Pure helpers:
  - `autoDimension.ts::generateAutoDimensionChain(elementsById, ctx, axis)`
    finds walls perpendicular to the chain axis, dedupes endpoint
    positions, and emits a chain of `createDimension` commands.
  - `autoTags.ts::generateAutoTags(elementsById, ctx, kind)` walks
    rooms / doors / windows, computes centroids (polygon centroid for
    rooms; alongT-projected wall point for openings), and emits
    `placeTag` commands. Includes `clearAutoAnnotationsCommand` for
    the clear-and-regenerate idempotency.
- `AnnotateRibbon.tsx` mounts top-right when a plan view is active:
  Auto Dim X / Auto Dim Y / Auto Tag — Rooms / Doors / Windows. Each
  button first emits `clearAutoGeneratedAnnotations` for the
  appropriate scope, then dispatches the helper-generated commands.

### ANN-01 — Detail components (`done`)

- Three new element kinds: `detail_line`, `detail_region`, `text_note`.
  Mirrored in `app/bim_ai/elements.py` with `createDetailLine`,
  `createDetailRegion`, `createTextNote` commands. Engine validates
  host view kind (`plan_view`/`section_cut`/`elevation_view`) and
  geometry minimums (≥2 line points, ≥3 boundary vertices, non-empty
  text).
- `detailComponentsRender.ts` exposes `extractDetailComponentPrimitives`,
  `svgTextAnchorAttrs`, `svgFillPatternDef` (hatch_45 / hatch_90 /
  crosshatch / dots SVG `<pattern>` defs), and `textNoteAnchorOffsetMm`
  for sheet renderers. Pure helpers + 11 unit tests.
- Plan-canvas three.js rendering: detail lines (LineDashed for
  `dashed`/`dotted`), detail regions via `THREE.ShapeGeometry`
  rotated onto the plan slice, text notes via `CanvasTexture` sprites
  scaled from `fontSizeMm`. The detail elements are skipped in 3D
  rendering by virtue of the `placed_tag`/`detail_*`/`text_note`
  kinds not being touched in any 3D mesh builder.
- Annotate ribbon also exposes Detail Line / Detail Region (Sketch) /
  Text Note buttons — for now they drop placeholder geometry at the
  view centre so rendering can be exercised end-to-end. In-canvas
  click-to-author is intentionally deferred until SKT-01 lands (per
  spec — "the load-bearing slice").
