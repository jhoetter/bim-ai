## Capabilities used

- **SKB-02** mass primitives in Phase 1 (deleted at the start of Phase 2 to avoid coplanar slab z-fights).
- **KRN-11** asymmetric gable roof: `ridgeOffsetTransverseMm: 1800`, `eaveHeightLeftMm: 1200`, `eaveHeightRightMm: 4500`, `slopeDeg: 45`. Slope sanity: `eaveLeft + leftRun·tan = 1200 + 4300·1 = 5500` above level → 1000 mm above east eave (avoids the seed-fidelity flat-roof failure).
- **KRN-12** trapezoidal loggia window with `outlineKind: 'gable_trapezoid'` + `attachedRoofId: 'hf-roof-main'`.
- **KRN-13** sliding-glass doors (`operationType: 'sliding_double'`) for the loggia + dormer-deck connections.
- **KRN-14** dormer on east slope (footprint 2400 × 2000, walls 2400, flat dormer roof, white_render). Sliding doors host on `hf-w-uf-e` in the cut footprint per dormerMesh's "+X open face" convention.
- **KRN-15** sweep along the south-face gable polygon (5-vertex pentagon, profile 100×200 mm, white_render).
- **KRN-16** wall recess on `hf-w-uf-s` (alongTStart=0.10, alongTEnd=0.90, setbackMm=1500). Wall primary `materialKey: 'cladding_warm_wood'` → recess back wall renders as wood, end caps auto-render white via `makeRecessedWallMesh`'s capMat override.
- **MAT-01** materials: `cladding_beige_grey`, `cladding_warm_wood`, `white_render`, `metal_standing_seam_dark_grey`, `aluminium_dark_grey`, `glass_clear`.
- **GAP-R4** balcony with frameless-glass balustrade, projection 400 mm.
- **SKB-16** camera presets — 4 viewpoints (vp-main-iso / vp-front-elev / vp-side-elev-east / vp-rear-axo).
- **VIE-04** persisted-viewpoint hud (auto on `setActiveViewpointId`).
- **VAL-01 / KRN-06** internal origin auto-created.

## Iterations spent per phase

Per skill 10-iteration cap — none of the 7 phases hit the cap.
Phase 3 (envelope) required one engine-API correction (the
`updateElementProperty` for roofs only supports `name | roofTypeId |
roofGeometryMode (mass_box | gable_pitched_rectangle)`, so
`asymmetric_gable` had to be authored via `deleteElement` +
`createRoof` rather than mutate-in-place). Phase 7 (documentation)
required one camera-application correction (`setActiveViewpointId`
flags the active viewpoint metadata; the camera move requires an
explicit `setOrbitCameraFromViewpointMm({ position, target, up })`
call — surfaced via the spec when the front-elev render was
identical to the iso default).
